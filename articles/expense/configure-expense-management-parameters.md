# <a name="configure-expense-management-parameters"></a>Налаштування параметрів керування витратами

_**Застосовується до:** Project Operations для сценаріїв на основі ресурсів і відсутності запасів_

У цій статті описано параметри елемента керування загальною поведінкою в управлінні витратами.

## <a name="general"></a>Основні

| Поле                                                    | Опис |
|----------------------------------------------------------|-------------|
| Стандартна ставка для пробігу                                 | Уведіть ставку для відшкодування витрат за пробіг. Ця ставка множиться на вказаний для витрати пробіг, і розраховується сума відшкодування для витрат на подорож. |
| Перевірка призначення витрат                                 | Увімкніть цей параметр, щоб під час створення звітів про витрати обмежити користувачів наявним набором значень, налаштованим в полі **Призначення звіту про витрати**. |
| Особисті витрати сплачуються                                | Виберіть відповідального за сплату будь-яких сум за транзакціями кредитної картки, які класифікуються як персональні. |
| Відображення усього звіту про витрати на деталізації               | Виберіть цей параметр, щоб показати всі витрати зі звіту про витрати при перегляді відомостей з вихідного документу для певного ваучера, створеного під час публікації звіту про витрати. |
| Попередня авторизація подорожі є обов'язковою                 | Виберіть цей параметр, щоб вимагати надсилання та затвердження заявок на подорожі перед поданням звіту про витрати. |
| Дозволити редагування розподілу перед публікацією            | Виберіть, чи можна редагувати розподіл звітів про витрати перед публікацією звіту. |
| Перевірка відповідності політикам керування витратами                     | Виберіть спосіб обчислення витрат, щоб визначити, чи порушується політика витрат. Витрати можуть оцінюватися з точки зору порушення політик під час введення та збереження витрати або після подання витрати на затвердження. Правила політик для квитанцій, що вимагаються, будуть перевірятися під час збереження рядка витрат. |
| Дозволити позиції витрат, що відбуваються у інших компаніях                         | Виберіть, чи можна вносити витрати для інших юридичних осіб у звіти про витрати. |
| Дозволити редагування обмінного курсу для витрат за кредитною карткою | Виберіть цей параметр, щоб дозволити користувачу змінювати обмінний курс для імпортованих витрат кредитної картки. |
| Багаторівневі поля ієрархії для відображення                  | Виберіть, які поля затверджувача відображаються, коли вказаний тип призначення робочого циклу є ієрархічним, а поле вибору **Ієрархія** налаштовано на використання ієрархії затверджень, багаторівневе затвердження витрат. Якщо для робочого циклу використовується багаторівнева ієрархія затвердження, вибрані поля відображатимуться в звіті про витрати та будуть доступні для редагування. |
| Введіть номер кредитної картки працівника                        | Виберіть, чи можна ввести та зберегти 15-значний або 16 значний номер в поле **Ідентифікатор картки** на сторінці **Кредитні картки** для працівника. |
| Перевірка призначення заявки для подорожі                      | Увімкніть цей параметр, щоб під час створення заявок на подорожі обмежити користувачів наявним набором значень, налаштованим в полі **Призначення звіту про витрати**. |

## <a name="financial"></a>Фінансова діяльність

| Поле                                                                                    | Опис |
|------------------------------------------------------------------------------------------|-------------|
| Ім'я щоденного журналу бухгалтерської книги                                                                | Виберіть ім'я журналу бухгалтерської книги, в якому публікуватимуться затверджені витрати. |
| Увімкнення податкового відшкодування за витратами                                                        | Виберіть цей параметр, щоб увімкнути податкове відшкодування для витрат, що підлягають такому відшкодуванню. Цей параметр не можна вибрати, якщо увімкнуто правила для податку на збут США та використання податків. |
| Негайна публікація готівкових авансів                                                           | Виберіть цей параметр, щоб публікувати затверджений готівковий аванс після завершення процесу сплати та передавання. Якщо цей параметр не вибрано, процес сплати та передавання створює неопублікований головний журнал. |
| Виправлення дати бухгалтерського обліку під час публікації                                                   | Виберіть цей параметр, щоб оновити дату бухгалтерського обліку під час публікації витрат, якщо поточний період призупинено або закрито. Дата бухгалтерського обліку буде встановлена на наступний відкритий період. |
| Дозволити групування транзакцій на основі компенсаційного рахунку, вказаного у способі оплати       | Виберіть цей параметр, щоб звести транзакції витрат у звіті про витрати на основі компенсаційного рахунку, вказаного у способі оплати для витрат. |
| Податок включено                                                                             | Виберіть цей параметр, щоб за замовчуванням додавати податок на збут до суми витрати. |
| Звільнення обтяжень при закритті заявок на подорожі                                      | Виберіть цей параметр, щоб звільняти суму обтяжень при закритті затвердженої заявки на подорож. |
| Дозволити подання заявок на подорожі при перевитраті бюджету за бюджетним реєстром та бюджетом проекту | Виберіть цей параметр, щоб працівники могли подавати на затвердження заявки на подорожі, навіть якщо вони перевищують дозволений бюджет, встановлений в бюджетному реєстрі, або бюджет проекту. |
| Дозволити подання звітів про витрати при перевитраті бюджету за бюджетним реєстром та бюджетом проекту     | Виберіть цей параметр, щоб працівники могли подавати на затвердження звіти про витрати, навіть якщо вони перевищують дозволений бюджет, встановлений в бюджетному реєстрі або бюджет проекту. |
| Дозволити затвердження звітів про витрати при перевитраті бюджету тільки за бюджетним реєстром                 | Виберіть цей параметр, щоб дозволити затверджувачам затверджувати звіти про витрати, які перевищують дозволений бюджет, встановлений у бюджетному реєстрі. |

## <a name="per-diem"></a>Добові

| Поле                                 | Опис |
|---------------------------------------|-------------|
| Мінімальна кількість годин для добових            | Введіть мінімальну кількість годин за замовчуванням, яку працівник повинен відпрацювати за день, щоб мати право на отримання добових для витрат, пов'язаних із подорожами. Це значення використовується як значення за замовчуванням лише для поля **Мінімальна кількість годин** для рівнів добових ставок. |
| Відсоток на харчування                          | Уведіть відсоток за замовчуванням для добових на харчування, який використовується в перший та останній день для витрат, пов'язаних із подорожами, якщо в полі **Підрахунок зменшення харчування** встановлено значення **Тип харчування на добу** або **Кількість прийомів їжі за добу**. Робочий час для першого і останнього дня може бути коротшим за стандартний робочий день. Тому сума добових у ці дні може відрізнятись від стандартної суми. Якщо у відсотках встановлено значення **0** (нуль), то зменшення для першого і останнього днів становитиме 0,00. |
| Відсоток на готель                         | Введіть відсоток добових за замовчуванням для готелів, що використовується для першого і останнього дня витрат, пов'язаних із подорожами. Робочий час для першого і останнього дня може бути коротшим за стандартний робочий день. Тому сума добових у ці дні може відрізнятись від стандартної суми. Якщо у відсотках встановлено значення **0** (нуль), то зменшення для першого і останнього днів становитиме 0,00. |
| Відсоток на інше                         | Введіть відсоток добових за замовчуванням для інших витрат, що використовується для першого і останнього дня витрат, пов'язаних із подорожами. Робочий час для першого і останнього дня може бути коротшим за стандартний робочий день. Тому сума добових у ці дні може відрізнятись від стандартної суми. Якщо у відсотках встановлено значення **0** (нуль), то зменшення для першого і останнього днів становитиме 0,00. |
| Зменшення у відсотках для сніданку | Уведіть суму, на яку буде зменшено добові для сніданку. Наприклад, якщо працівник отримує безкоштовний сніданок, можливо, ви захочете зменшити суму добових на 10 відсотків. |
| Зменшення у відсотках для обіду     | Уведіть суму, на яку буде зменшено добові для обіду. Наприклад, якщо працівник отримує безкоштовний обід, можливо, ви захочете зменшити суму добових на 15 відсотків. |
| Зменшення у відсотках для вечері    | Уведіть суму, на яку буде зменшено добові для вечері. Наприклад, якщо працівник отримує безкоштовну вечерю, можливо, ви захочете зменшити суму добових на 25 відсотків. |
| Розрахунок зменшень на харчування           | Укажіть для системи спосіб розрахунку зменшень на харчування, вибравши, чи залежать зменшення від типу харчування для поїздки або кожного дня, або ж вони залежать від дозволеної кількості прийомів їжі на добу. |
| Округлення добових                     | Виберіть тип округлення, який використовуватиметься для витрат на добові. Якщо вибрати нормальне округлення, то будь-які витрати добових, які мають суму 0,50, округлюватимуться до 1,00, а витрати добових, менші за 0,50, округлюватимуться до 0,00. |
| Період для базового розрахунку добових          | Виберіть, чи сума добових розраховується для календарних днів або ж для 24-годинного періоду. |

## <a name="fax-cover-pages"></a>Титульні сторінки факсу

| Поле                          | Опис |
|--------------------------------|-------------|
| Інструкції                   | Укажіть інструкції, які мають виконувати працівники під час створення титульної сторінки для факсу, що використовуватиметься для відправки квитанцій, пов'язаних із звітом про видатки. Щоб ввести текст, який залежатиме від мови користувача, виберіть **Переклади**. |
| Ідентифікатор користувача (відомості про штрих-код) | Виберіть цей параметр, щоб зберігати унікальний ідентифікатор працівника у штрих-коді, який використовуватиметься на титульній сторінці факсу. |
| Штрих-код                       | Виберіть штрих-код, який буде використовуватись на титульній сторінці факсу. У Керуванні витратами підтримуються штрих-коди 39 і 128. |

## <a name="anti-corruption"></a>Протидія корупції

| Поле                                 | Опис |
|---------------------------------------|-------------|
| Показати антикорупційну атестацію   | Виберіть цей параметр, щоб показувати антикорупційний текст при створенні звіту про витрати. Можна ввімкнути певні категорії витрат, для яких необхідно вибрати антикорупційну атестацію у звіті про витрати. Наприклад, категорія подарунків, пов'язана з офіційними урядовими витратами, може вимагати від працівника підтвердження того, що витрата відповідає політиці компанії, пов'язаній із офіційними представниками уряду. |
| Антикорупційне повідомлення для подавця | Уведіть текст, який має побачити працівник, котрий створює звіт про витрати. Щоб ввести текст, який залежатиме від мови користувача, виберіть **Переклади**. |
| Антикорупційне повідомлення для затверджувача  | Уведіть текст, який має побачити особа, що затверджує звіт, коли буде створено звіт про видатки. Щоб ввести текст, який залежатиме від мови користувача, виберіть **Переклади**. |
