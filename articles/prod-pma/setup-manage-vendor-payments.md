# <a name="set-up-and-use-pay-when-paid-vendor-payments"></a><span data-ttu-id="4bf78-103">Налаштування та використання оплати для постачальників у режимі «оплата після оплати»</span><span class="sxs-lookup"><span data-stu-id="4bf78-103">Set up and use pay-when-paid vendor payments</span></span>

[!include [banner](../includes/banner.md)]

<span data-ttu-id="4bf78-104">У разі схвалення постачальника для роботи в якості субпідрядника, можливо, потрібно буде затримати оплату для постачальника, доки клієнт не виплатить кошти за проект.</span><span class="sxs-lookup"><span data-stu-id="4bf78-104">When you approve a vendor to work as a subcontractor, you might want to withhold payment to the vendor until your customer pays you for the project.</span></span> <span data-ttu-id="4bf78-105">Щоб підтримати цей сценарій, можна налаштувати умови оплати після оплати під час налаштування замовлення на придбання з постачальником.</span><span class="sxs-lookup"><span data-stu-id="4bf78-105">To support this scenario, you can set up pay-when-paid (PWP) terms when you set up the purchase order (PO) with the vendor.</span></span>

<span data-ttu-id="4bf78-106">Наприклад, коли клієнт сплачує суму за рахунком проекту, можна частково або повністю виділити суму для оплати рахунка постачальника.</span><span class="sxs-lookup"><span data-stu-id="4bf78-106">For example, when a customer pays an amount on a project invoice, you can release some or all of the vendor invoice amount.</span></span> <span data-ttu-id="4bf78-107">Просто налаштуйте умови оплати після оплати, згідно з якими постачальник отримає оплату після отримання відсотка відповідного платежу від клієнта.</span><span class="sxs-lookup"><span data-stu-id="4bf78-107">Just set up PWP terms that specify that the vendor will be paid after you receive a percentage of the related payment from the customer.</span></span> <span data-ttu-id="4bf78-108">У разі отримання часткової оплати від клієнта можна вручну виділити суму для оплати певних рахунків відповідного постачальника.</span><span class="sxs-lookup"><span data-stu-id="4bf78-108">If you receive partial payment from a customer, you can manually release some of the related vendor invoices for payment.</span></span>

<span data-ttu-id="4bf78-109">У прикладі нижче розглянуто процес, коли використовуються умови оплати після оплати.</span><span class="sxs-lookup"><span data-stu-id="4bf78-109">The following example outlines the process when PWP terms are used.</span></span>

## <a name="example"></a><span data-ttu-id="4bf78-110">Приклад</span><span class="sxs-lookup"><span data-stu-id="4bf78-110">Example</span></span>

<span data-ttu-id="4bf78-111">Ваша організація зобов’язується надати клієнту 100 комп’ютерів з інстальованим програмним забезпеченням за ціною 150,00 доларів США за комп’ютер.</span><span class="sxs-lookup"><span data-stu-id="4bf78-111">Your organization agrees to provide a customer with 100 computers that have software installed, for a price of 150.00 US dollars (USD) per computer.</span></span> <span data-ttu-id="4bf78-112">Потім ви наймаєте постачальника, який надасть вам комп’ютери з інстальованим програмним забезпеченням.</span><span class="sxs-lookup"><span data-stu-id="4bf78-112">You then hire a vendor to provide you with the computers that have software installed.</span></span> <span data-ttu-id="4bf78-113">Згідно з угодою, клієнт повинен схвалити якість комп’ютерів, перш ніж сплатити кошти вашій організації.</span><span class="sxs-lookup"><span data-stu-id="4bf78-113">According to the agreement, the customer must approve the quality of the computers before it pays your organization.</span></span> <span data-ttu-id="4bf78-114">Політика вашої організації полягає в тому, щоб затримувати оплату для постачальників, доки клієнт не сплатить вам кошти.</span><span class="sxs-lookup"><span data-stu-id="4bf78-114">Your organization's policy is to withhold payment to vendors until the customer has paid you.</span></span> <span data-ttu-id="4bf78-115">Таким чином, можна налаштувати проект так, щоб відсоток оплати після оплати становив 100 відсотків.</span><span class="sxs-lookup"><span data-stu-id="4bf78-115">Therefore, you set up the project so that it has a PWP percentage of 100 percent.</span></span>

<span data-ttu-id="4bf78-116">Постачальник надсилає вам 100 комп’ютерів з інстальованим програмним забезпеченням разом із рахунком на 10 000,00 доларів США.</span><span class="sxs-lookup"><span data-stu-id="4bf78-116">The vendor sends you the 100 computers that have software installed, together with an invoice for 10,000.00 USD.</span></span> <span data-ttu-id="4bf78-117">Оскільки для вашого проекту налаштовано умови оплати після оплати, ви не платите постачальнику під час отримання комп’ютерів.</span><span class="sxs-lookup"><span data-stu-id="4bf78-117">Because PWP terms are set up for your project, you don't pay the vendor upon receipt of the computers.</span></span> <span data-ttu-id="4bf78-118">Замість цього ви надсилаєте комп’ютери клієнту разом із рахунком на 15 000,00 доларів США.</span><span class="sxs-lookup"><span data-stu-id="4bf78-118">Instead, you send the computers to the customer, together with an invoice for 15,000.00.</span></span> <span data-ttu-id="4bf78-119">Клієнт перевіряє комп’ютери й затверджує повну суму рахунка проекту.</span><span class="sxs-lookup"><span data-stu-id="4bf78-119">The customer inspects the computers and approves the full amount of the project invoice.</span></span>

<span data-ttu-id="4bf78-120">Після отримання повної оплати від клієнта ви сплачуєте постачальнику 10 000,00 доларів США — повну суму рахунка постачальника.</span><span class="sxs-lookup"><span data-stu-id="4bf78-120">After you receive the full payment from the customer, you pay the vendor 10,000.00, the full amount of the vendor invoice.</span></span>

## <a name="set-up-pwp-terms-for-a-project"></a><span data-ttu-id="4bf78-121">Налаштування умов оплати після оплати для проекту</span><span class="sxs-lookup"><span data-stu-id="4bf78-121">Set up PWP terms for a project</span></span>

<span data-ttu-id="4bf78-122">Під час налаштування умов оплати після оплати для проекту необхідно вказати у відсотках мінімальну суму, яку клієнт має сплатити за проект, перш ніж ви зможете виплатити кошти постачальнику.</span><span class="sxs-lookup"><span data-stu-id="4bf78-122">When you set up PWP terms for a project, you must specify, as a percentage, the minimum amount that a customer must pay you for the project before you will pay the vendor.</span></span> <span data-ttu-id="4bf78-123">Порогова сума автоматично обчислюється для рахунків клієнта для проекту.</span><span class="sxs-lookup"><span data-stu-id="4bf78-123">The threshold amount is automatically calculated on the customer invoices for the project.</span></span> <span data-ttu-id="4bf78-124">Виконайте кроки нижче, щоб налаштувати пороговий відсоток для умов оплати після оплати.</span><span class="sxs-lookup"><span data-stu-id="4bf78-124">Follow these steps to set up the threshold percentage for PWP terms.</span></span>

1. <span data-ttu-id="4bf78-125">Перейдіть до розділу **Керування проектом та бухгалтерський облік** \> **Проекти** \> **Усі проекти**.</span><span class="sxs-lookup"><span data-stu-id="4bf78-125">Go to **Project management and accounting** \> **Projects** \> **All projects**.</span></span>

3. <span data-ttu-id="4bf78-127">На швидкій вкладці **Угоди постачальника** виберіть **Додати рядок**.</span><span class="sxs-lookup"><span data-stu-id="4bf78-127">On the **Vendor agreements** FastTab, select **Add line**.</span></span>
3. <span data-ttu-id="4bf78-128">У полі **Код облікового запису** виберіть один із варіантів нижче:</span><span class="sxs-lookup"><span data-stu-id="4bf78-128">In the **Account code** field, select one of the following options:</span></span>

    - <span data-ttu-id="4bf78-130">**Група** — умови оплати після оплати застосовуються до всіх постачальників у групі постачальників.</span><span class="sxs-lookup"><span data-stu-id="4bf78-130">**Group** – The PWP terms apply to all vendors in a vendor group.</span></span>
    - <span data-ttu-id="4bf78-131">**Усі** — умови оплати після оплати застосовуються до всіх постачальників.</span><span class="sxs-lookup"><span data-stu-id="4bf78-131">**All** – The PWP terms apply to all vendors.</span></span>

4. <span data-ttu-id="4bf78-132">Якщо в попередньому кроці вибрано параметр **Таблиця** або **Група**, у полі **Постачальник/група постачальників** виберіть постачальника або групу постачальників, до яких будуть застосовуватися умови оплати після оплати.</span><span class="sxs-lookup"><span data-stu-id="4bf78-132">If you selected **Table** or **Group** in the previous step, in the **Vendor/Vendor group** field, select the vendor or vendor group that the PWP terms apply to.</span></span> <span data-ttu-id="4bf78-133">Якщо в попередньому кроці вибрано параметр **Усі**, поле **Постачальник/група постачальників** неможливо редагувати.</span><span class="sxs-lookup"><span data-stu-id="4bf78-133">If you selected **All** in the previous step, the **Vendor/Vendor group** field can't be edited.</span></span>
5. <span data-ttu-id="4bf78-134">Якщо для постачальника в проекті налаштовано умови затримання оплати для постачальника, у полі **Умови затримання оплати для постачальника** виберіть ідентифікатор правила для умов затримання оплати.</span><span class="sxs-lookup"><span data-stu-id="4bf78-134">If vendor retention terms are set up for the vendor in the project, in the **Vendor retention terms** field, select the rule ID for the retention terms.</span></span>
6. <span data-ttu-id="4bf78-135">У полі **Пороговий відсоток оплати після оплати** введіть пороговий відсоток для проекту.</span><span class="sxs-lookup"><span data-stu-id="4bf78-135">In the **PWP threshold percentage** field, enter the threshold percentage for the project.</span></span> <span data-ttu-id="4bf78-136">Відсоток, який уводиться для проекту, визначає мінімальну суму, яку клієнт має сплатити, перш ніж ви зможете сплатити послуги постачальника.</span><span class="sxs-lookup"><span data-stu-id="4bf78-136">The percentage that you enter for the project defines the minimum amount that the customer must pay you before you will pay the vendor.</span></span>

## <a name="create-a-po-that-has-pwp-terms"></a><span data-ttu-id="4bf78-137">Створення замовлення на придбання з умовами оплати після оплати</span><span class="sxs-lookup"><span data-stu-id="4bf78-137">Create a PO that has PWP terms</span></span>

1. <span data-ttu-id="4bf78-140">Перейдіть до розділу **Закупівлі та постачання** \> **Замовлення на придбання** \> **Усі замовлення на придбання**.</span><span class="sxs-lookup"><span data-stu-id="4bf78-140">Go to **Procurement and sourcing** \> **Purchase orders** \> **All purchase orders**.</span></span>
