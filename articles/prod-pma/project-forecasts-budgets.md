# <a name="project-forecasts-and-budgets"></a>Прогнози та бюджети проекту

[!include [banner](../includes/banner.md)]

Існує два способи для керування та контролю вашими проектами: прогнози та бюджети проекту. 

Використовуйте прогнозування проекту, якщо в організації є операційна перспектива, і якщо вона зосереджується на доходах і витратах, отриманих із певних транзакцій. Використовуйте бюджет проекту, якщо ваша організація більше зосереджена на фінансових сумах. 

Прогнози та бюджети проекту використовують моделі прогнозу для збереження проектованих сум транзакцій. 

Кожен метод має свої переваги. Перш ніж вибрати метод для організації, потрібно врахувати зазначені нижче моменти.

|   Спосіб виділення       |           Прогнозування проекту            |        Бюджет проекту                           |
|---------------------------|------------------------------------------|----------------------------------------------------|
| **Розподіл за періодом**     | Не можна явно виділяти транзакції у звітний період. Натомість прогноз, а також контроль за прогнозами, ґрунтується на життєвому циклі проекту. Оскільки прогнози базуються на певній даті, необхідно зробити цей період з дати. | Транзакції можна виділити за весь період або звітний період. Якщо ви виділяєте за період, ви можете перенести невикористані суми на наступний звітний період. |
| **Перегляд транзакцій**  | Транзакції можна переглянути в прогнозованих формах, де відображаються прогнози для всієї компанії та для всіх проектів, незалежно від ієрархії. Щоб зосередитися на конкретному проекті, слід відфільтрувати дані.                                       | Для однієї ієрархії проекту можна переглянути транзакції з бюджетом. Тому можна переглянути відомості про транзакції для первинного проекту або його підпроектів.                 |
| **Змінні транзакції** | Під час введення транзакцій прогнозування можна використовувати кожен атрибут, який існує для фактичної транзакції. Це дає змогу отримати більше подробиць у прогнозі. Наприклад, можна ввести відомості для кількості, працівників, елементів або властивостей рядка.         | Під час введення відомостей про бюджет можна використовувати лише суми, категорії та справи.                    |
| **Безпека**              | Прогнозування ґрунтується на транзакціях, введених у формах прогнозів і не передбачає механізму керування процесом. Будь-який працівник, який має дозволи для форми прогнозу, може переглянути інформацію без схвалення.                                        | У бюджеті використовується система робочих циклів, яка дає змогу керувати змінами та відстежувати журнал змін.         |
| **Типи запису**           | Записи прогнозованих транзакцій визначаються за кількістю одиниць вимірювання, а також за вартістю та за ціною одиниці збуту.  | Інформація про бюджет залежить від сум, які розподіляються між витратами та доходами.                                          |
| **Моделі прогнозування**       | Оскільки кожен прогноз має бути зв'язаний із моделлю, можна створити кілька моделей прогнозування, а також налаштувати вкладені моделі.           | Проектне бюджетування обмежує моделі прогнозів, які використовуються для бюджетування. Менше моделей прогнозів можуть сприяти підвищенню узгодженості в прогнозах.                           |
| **Переповнення вартості**         | Можна лише дозволити або заборонити запис транзакцій, який призведе до переповнення витрат.   | Бюджетування проекту надає додаткові параметри керування для користувачів. Можна дозволити попередження та переповнення.                    |
| **Елемент керування**               | Контроль за прогнозами здійснюється шляхом зменшення прогнозування. Фактичні суми вираховуються від прогнозованих залишків без будь-яких контрольного журналу. Це може ускладнити відстеження, де відбулися фактичні транзакції.                   | У бюджетному контролі за проектом фактична сума віднімається від суми в бюджеті, що залишився. Це дасть змогу вести більш чіткий контрольний журнал.                                   |

## <a name="project-forecasts"></a>Прогнози проекту
У разі використання прогнозування проекту можна ввести прогнозовані транзакції у формах прогнозу для кожного типу транзакції. Кожен атрибут, доступний для фактичної транзакції, можна використовувати для прогнозної транзакції — наприклад, рентабельність рядка, атрибути рядка, працівники або описи. Крім того, можна спрогнозувати, за який час після понесення витрат ви будете виставляти рахунок клієнту. 

Операції з прогнозування проекту базуються на одиницях і сумах. 

За допомогою моделі прогнозів потрібно зв'язати кожен прогноз проекту. Під час введення прогнозованої транзакції автоматично пропонується модель прогнозу. Модель прогнозування виконує роль контейнера для прогнозованих транзакцій. 

Моделі прогнозу можна визначити як підмоделі для моделі. Це дасть змогу прогнозувати за регіоном, періодом часу або відділом. Можна скопіювати прогнозовані транзакції у модель, щоб створити нову модель, а також можна перенести транзакції до загальної бухгалтерської книги. Оскільки між прогнозом і моделлю існує тісний зв'язок, модель кожного прогнозу становить окремий бюджет для проекту. 

Моделі прогнозів можуть використовувати зниження прогнозу в ролі механізму керування проектами. У разі зменшення прогнозу фактичні транзакції зменшують прогнозовані залишки транзакцій. Проте, оскільки зменшення прогнозу застосовується до найвищого проекту в ієрархії, воно забезпечує обмежене представлення змін у прогнозі. Наприклад, якщо працівник зв'язаний із підпроектом, фактичні транзакції для працівника надсилаються до первинного проекту. Це може ускладнити відстеження змін, оскільки не можливо легко визначити, яка транзакція у якому підпроекті спричинила зменшення суми прогнозу. Тому рекомендовано створити копію моделі прогнозу для використання при зменшенні прогнозу. Потім можна використати звіти, щоб переглянути, що спочатку було прогнозовано. 

Можна переглянути, копіювати, видаляти або передавати прогнози проекту на загальний бюджет бухгалтерської книги. Однак не можна керувати процесом. Будь-який працівник, який має дозвіл для форми прогнозу, може переглянути інформацію без відгуку.

-   **Перевірити** – можна перевірити прогнозовані транзакції в одних і тих самих формах, де було зроблено вихідні записи.
-   **Копіювати або видалити** — під час копіювання прогнозованих транзакцій можна копіювати рядки транзакції з однієї моделі прогнозу до іншої моделі. У разі видалення прогнозу буде видалено прогнозовані транзакції з моделі прогнозу. Щоб обмежити скопійовані або видалені прогнозовані транзакції, виберіть певні типи транзакцій і дати. Це дасть змогу копіювати або видаляти лише певну частину прогнозу.
-   **Передати** — коли ви передаєте прогноз проекту до загального бюджету бухгалтерської книги, ви передаєте прогнозні транзакції моделі прогнозу до загального бюджету бухгалтерської книги. Будь-який раніше передані транзакції можна перезаписати в бюджеті бухгалтерської книги, в яку ви переносите прогноз проекту.

## <a name="project-budgets"></a>Бюджети проекту
Створення бюджету проекту — це простіший метод, ніж прогнозування, хоча воно інтегрується з моделями прогнозу. У ньому використовується єдина форма вводу для отримання вихідних відомостей і змін бюджету, а також дає змогу робити прогнози, які базуються лише на сумі, категорії або справі. 

У рамках створення бюджету проекту всі вихідні бюджети та виправлення мають бути надіслані до робочого циклу проекту для ухвалення. Робочі цикли дають змогу збільшити контроль над процесом, а також створити запис журналу змін. 

Створення бюджету проекту схоже на створення загального бюджету бухгалтерської книги, але його швидше і простіше налаштувати. Більшість варіантів складання загального бюджету, наприклад числових послідовностей або грошової одиниці, не обов'язково встановлюються окремо для проектів.

Бюджети проекту автоматично зв'язані з двома прогнозними моделями, по одному для вихідного бюджету та для решти бюджету. Тому звіти, що базуються на моделях прогнозування, можуть використовувати бюджетні дані. Коли бюджет проекту підтверджений, система створює прогнозовані транзакції на основі пов'язаних моделей, які використовуються для звітування та керування цілями.

## <a name="forecast-models"></a>Моделі прогнозу
Моделі прогнозів мають одношарову ієрархію. Це означає, що один прогноз проекту має бути зв'язаний із однією моделлю прогнозу.

Якщо ви використовуєте прогнозування проекту, можна визначити моделі як вкладені моделі. Потім ви можете створити прогнози за відділами, періодом часу або регіоном. Наприклад, можна створити модель прогнозу на рік, а потім створити підмоделі для північно-східного, південно-західного та південно-західного регіонів, які будуть представлені регіональними керівниками. Якщо вибрати різні параметри в доступних звітах, можна переглянути відомості за загальним прогнозом або за вкладеною моделлю.
