# <a name="financial-estimation-concepts"></a>Концепції фінансової оцінки

_**Застосовується до:** Project Operations для сценаріїв на основі ресурсів і відсутності запасів, полегшене розгортання: угоди та виставлення рахунків-проформ_

У Dynamics 365 Project Operations ви можете виконати фінансову оцінку своїх проектів у два етапи. 
1. На етапі попереднього збуту до укладення угоди. 
2. На етапі виконання після створення сервісного договору проекту. 

Ви можете створити фінансову оцінку для роботи за проектом із використанням будь-якої з наступних трьох сторінок.
- На сторінці **Позиції в ціновій пропозиції** з використанням відомостей позиції цінової пропозиції.  
- На сторінці **Сервісна робота за проектним договором** із використанням відомостей позиції цінової пропозиції. 
- На сторінці **Проект** із використанням сторінок вкладок **Завдання** або **Кошториси витрат**.

## <a name="use-a-project-quote-to-create-an-estimate"></a>Для створення кошторису користуйтеся проектною ціновою пропозицією
Для проектної цінової пропозиції ви можете використовувати сутність **Відомості позиції цінової пропозиції** для прогнозування роботи, необхідної для виконання проекту. Потім можна надіслати ці прогнози клієнтові.

Позиції проектної цінової пропозиції можуть мати записи із відомостями щодо певної позиції цінової пропозиції, або не мати їх. Відомості позиції цінової пропозиції використовуються для оцінки часу, витрат або комісій. Програма Microsoft Dynamics 365 Project Operations не забезпечує можливість складати прогнози щодо матеріалів за відомостями позиції цінової пропозиції. Вони називаються класами транзакцій. Прогнозовані суми податку також можуть бути введені в класі транзакцій.

На додачу до класів транзакції, відомості позиції цінової пропозиції мають тип транзакції. Підтримуються два типи операцій для відомостей позиції цінової пропозиції: **Вартість** і **Проектний договір**.

## <a name="use-a-project-contract-to-create-an-estimate"></a>Для створення кошторису користуйтеся проектним сервісним договором

Якщо під час створення сервісного договору на основі проекту було використано цінову пропозицію, то прогноз для кожної позиції цінової пропозиції буде скопійовано до проектного договору. Структура проектного договору подібна до структури цінової пропозиції, яка містить позиції, відомості позиції та розклади виставлення рахунків.

Прогнози можна виконати безпосередньо в проектному договорі, як і в ціновій пропозиції проекту. Для цінової пропозиції прогноз здійснюється за допомогою сервісних робіт за договором і відомостей про сервісну роботу за договором. Відомості про сервісну роботу за договором також можна генерувати із плану проекту, який було створено за допомогою прогнозу за принципом «знизу догори».

Дані сервісної роботи за договором можна використовувати для оцінки часу, витрат або комісій. Прогнозована сума податків також може бути введена у відомості сервісної роботи за договором.

Матеріальні прогнози не підтримуються для відомостей сервісної роботи за договором.

## <a name="use-a-project-to-create-an-estimate"></a>Для створення кошторису користуйтеся проектом 

Ви можете прогнозувати час і витрати у проекті. Програма Project Operations не підтримує кошториси матеріалів або плату за проектами

Під час створення завдання та визначення атрибутів загального ресурсу, необхідного для виконання завдання, створюються прогнози часу. Часові прогнози створюються на основі завдань розкладу. У разі створення загальних учасників робочої групи за межами розкладу, прогнози часу не створюються.

Прогнози витрат вводяться в сітку на сторінці **Кошториси витрат**.

Найкращою практикою вважається створення кошторису для проекту, тому що є можливість створювати докладні колегіальні кошториси робочої сили або часу й витрат для кожного завдання в проектному плані. Після цього ви можете користуватися детальним кошторисом для створення кошторисів для кожної позиції цінової пропозиції, а також розробити реальнішу цінову пропозицію для клієнта. Коли ви імпортуєте або створюєте докладний кошторис у позиції цінової пропозиції з використанням плану проекту, Project Operations імпортує значення збуту й значення вартості цих кошторисів. Після імпортування ви можете переглянути показники рентабельності, коефіцієнти дохідності й техніко-економічні показники за ціновою пропозицією проекту.

## <a name="customize-the-quote-line-detail-and-contract-line-detail-entities"></a>Настроювання сутностей «Відомості про позицію цінової пропозиції» та «Відомості про сервісну роботу за договором»

Якщо було додано настроюване поле у відомості позиції цінової пропозиції і потрібно, щоб система вводила значення цього поля як значення за замовчуванням у відповідному рядку витрат, використовуйте реєстраційні засоби компонента plug-in **PreOperationContractLineDetailUpdate** та **PreOperationQuoteLineDetailUpdate**. Ці компоненти Plug-in потрібно повторно реєструвати після того, як докладно буде змінено позицію цінової пропозиції або відомості про сервісну роботу за договором. Щоб завершити процес, виконайте вказані нижче кроки.

1. Відкрийте PluginRegistrationTool і підключіться до інсталяції в онлайновому режимі.
2. Виберіть **Пошук**, а потім знайдіть компонент Plug-in для оновлення.
3. Виберіть компонент plug-in, а потім на основній сторінці клацніть **Вибрати**.
4. Виберіть крок компонента plug-in для оновлення, клацніть правою кнопкою миші, а потім виберіть **Оновити**.
5. У діалоговому вікні **Оновлення наявного кроку** в полі **Атрибути фільтрації** натисніть кнопку "три крапки" (**...**):
6. У діалоговому вікні **Вибрати атрибути** встановіть прапорці поруч із настроюваними атрибутами.
7. Щоб закрити діалогове вікно, натисніть кнопку **ОК**, а потім виберіть **Оновити крок**.
8. Повторіть кроки 1 – 7 для другого компонента plug-in.
9. Закрийте **PluginRegistrationTool**.
