# <a name="troubleshoot-working-in-the-task-grid"></a>Усунення проблем із роботою в сітці завдань 


_**Застосовується до:** Project Operations для сценаріїв на основі ресурсів і відсутності запасів, Розгортання Lite: від угоди до рахунків-проформ, Project for the web_

Сітка завдань, що використовується Dynamics 365 Project Operations — це розміщений iframe у Microsoft Dataverse. У результаті цього використання для забезпечення правильної функціональності автентифікації та авторизації слід виконати конкретні вимоги. У цій статті окреслюються поширені проблеми, які можуть впливати на спроможність відображення сітки або керування завданнями в робочій структурі проекту (WBS).

Поширені проблеми включають:

- Вкладка **Завдання** на сітці Завдань є порожньою.
- При відкритті проекту проект не завантажується, а користувацький інтерфейс (UI) зупиняється на блоці зі змінним значенням.
- Адміністрування правами для **Project for the Web**.
- Зміни не зберігаються під час створення, оновлення або видалення завдання.

## <a name="issue-the-task-tab-is-empty"></a>Проблема: вкладка «Завдання» порожня

### <a name="mitigation-1-enable-cookies"></a>Заходи з пом'якшення 1: увімкніть файли cookie

Project Operations вимагає увімкнення сторонніх файлів cookie для відображення робочої структури проекту. Якщо не ввімкнути файли cookie сторонніх виробників, замість завдань з’являться пусті сторінки, якщо вибрати вкладку **Завдання** на сторінці **Проект**.

У наведених процедурах описано, як у браузерах Microsoft Edge або Google Chrome оновити настройку браузера, щоб увімкнути файли сookie сторонніх постачальників.

#### <a name="microsoft-edge"></a>Microsoft Edge

1. Відкрийте браузер Microsoft Edge.
2. У верхньому правому куті натисніть **три крапки** (...), а потім натисніть **Настройки**.
3. У розділі **Файли сookie і дозволи для сайту** натисніть **Файли сookies і дані сайту**.
4. Вимкніть **Блокування файлів cookie від сторонніх постачальників**.
5. Оновіть сторінку в браузері. 

#### <a name="google-chrome"></a>Google Chrome

1. Відкрийте браузер Chrome.
2. У верхньому правому куті виберіть три вертикальні крапки а потім натисніть **Настройки**.
3. У розділі **Конфіденційність і безпека** натисніть **Файли сookie і інші дані сайту**.
4. Натисніть **Дозволити усі файли cookie**.
5. Оновіть сторінку в браузері. 

> [!NOTE]
> Якщо блокувати файли cookie сторонніх постачальників, усі файли cookie та дані сайтів з інших сайтів буде заблоковано, навіть якщо це дозволено списком винятків.

### <a name="mitigation-2-validate-the-pex-endpoint-has-been-correctly-configured"></a>Засіб пом’якшення 2: Переконатися в тому, що кінцева точка PEX налаштована правильно

Для Project Operations потрібне посилання на кінцеву точку PEX для параметра проекту. Ця кінцева точка потрібна для обміну інформацією зі службою, яка використовується для відображення робочої структури проекту. Якщо параметр не ввімкнено, ви отримаєте повідомлення про помилку «Неприпустимий параметр проекту». Щоб оновити кінцеву точку PEX, виконайте наведені далі кроки.

1. Додайте поле **Кінцева точка PEX** на сторінку **Параметри проекту**.
2. Визначте тип продукту, який ви використовуєте. Це значення використовується, якщо встановлено кінцеву точку PEX. Після отримання тип продукту вже визначено в кінцевій точці PEX. Збережіть це значення.
3. Оновіть поле з наведеним значенням: `https://project.microsoft.com/<lang>/?org=<cdsServer>#/taskgrid?projectId=<id>&type=2`. У наведеній далі таблиці викладено типи параметрів, які мають використовуватися, виходячи з типу продукту.

      | **Тип продукту**                     | **Параметр типу** |
      |--------------------------------------|--------------------|
      | Project for the Web в організації за замовчуванням   | type=0             |
      | Project for the Web в організації з іменем CDS | type=1             |
      | Project Operations                   | type=2             |

4. Видаліть поле зі сторінки **Параметри проекту**.

### <a name="mitigation-3-sign-in-to-projectmicrosoftcom"></a>Кроки з усунення 3: увійдіть в обліковий запис на сторінці project.microsoft.com

Відкрийте у браузері нову вкладку, перейдіть на project.microsoft.com і ввійдіть в обліковий запис за допомогою ролі користувача, яка використовується для доступу до Project Operations. Важливо, щоб у продукт Microsoft у браузері ввійшов лише один користувач. Повідомлення про помилку «login.microsoftonline.com відмовив у підключенні» найчастіше відображається, якщо вхід виконав більш, ніж один користувач, як показано на рисунку нижче.

![Виберіть сторінку входу до облікового запису, на якій з’являться два користувачі, які ввійшли в обліковий запис.](media/MULTIPLE_USERS_LOGGED_IN.png)

## <a name="issue-the-project-doesnt-load-and-the-ui-is-stuck-on-the-spinner"></a>Проблема: проект не завантажується, а користувацький інтерфейс (UI) зупиняється на блоці зі змінним значенням

Для цілей автентифікації необхідно увімкнути спливаючі вікна, щоб завантажилася сітка Завдань. Якщо спливаючі вікна не увімкнені, екран зупинятиметься на індикаторі завантаження. На наведеному далі рисунку показано URL-адресу із заблокованим спливаючим надписом у рядку адреси, що призводить до залипання індикатора завантаження при спробі завантаження сторінки. 

   ![Індикатор завантаження та спливаючий блок, що застрягли.](media/popupsblocked.png)

### <a name="mitigation-1-enable-pop-ups"></a>Заходи з пом'якшення 1: увімкніть спливаючі вікна

Коли ваш проект застрягає на індикаторі завантаження, можливо, що спливаючі вікна не увімкнені.

#### <a name="microsoft-edge"></a>Microsoft Edge

У браузері Edge можна увімкнути спливаючі вікна двома способами.

1. У браузері Edge виберіть сповіщення у верхній правій частині браузера.
2. Виберіть **Завжди дозволяти спливаючі вікна та переспрямування від** конкретного середовища Dataverse.
 
     ![Вікно заблокованих спливаючих вікон.](media/enablepopups.png)

Окрім цього, ви можете виконати наведені далі кроки.

1. Відкрийте браузер Microsoft Edge.
2. У правому верхньому куті виберіть **три крапки** (...), потім виберіть **Параметри** > **Дозволи сайту** > **Спливаючі вікна та переспрямування**.
3. Перемкніть **Спливаючі вікна та переспрямування** у положення «вимкнено» для блокування спливаючих вікон або перемкніть у положення «увімкнено», щоб дозволити їх на своєму пристрої.
4. Після увімкнення спливаючих вікон оновіть вікно браузера. 

#### <a name="google-chrome"></a>Google Chrome
1. Відкрийте браузер Chrome.
2. Перейдіть до сторінки, на якій блокуються спливаючі вікна.
3. У рядку адреси виберіть **Спливаючі вікна заблоковано**.
4. Виберіть посилання на спливаюче вікно, яке необхідно переглянути.
5. Після увімкнення спливаючих вікон оновіть вікно браузера. 

> [!NOTE]
> Щоб завжди бачити спливаючі вікна для сайту, виберіть пункт **Завжди дозволяти спливаючі вікна та переспрямування з [сайт]**, потім виберіть **Готово**.

## <a name="issue-3-administration-of-privileges-for-project-for-the-web"></a>Проблема 3: Адміністрування правами для Project for the Web

Project Operations залежать від зовнішньої служби планування. Служба вимагає призначення користувачу кількох ролей, що дозволяють користувачам виконувати зчитування та записування до сутностей, пов’язаних із WBS. До цих сутностей належать завдання проекту, призначення ресурсів і залежності завдань. Якщо користувач не може відобразити WBS при переході на вкладку **Завдання**, причина, швидше за все, полягає в тому, що **Проект** для **Project Operations** не увімкнений. Користувач може отримати повідомлення про помилку ролі безпеки або помилку, пов'язану з відмовою в доступі.

### <a name="mitigation-1-validate-the-application-user-and-end-user-security-roles"></a>Засіб пом'якшення 1: підтвердьте ролі безпеки користувача програми та кінцевого користувача

1. Перейдіть до розділу **Настройки** > **Безпека** > **Користувачі** > **Користувачі програми**.  

   ![Інструмент читання програми.](media/applicationuser.jpg)
   
2. Виконайте подвійне клацання на записі користувача програми, щоб переконатися в наведеному далі.

     - Користувач має доступ до проекту. Ви можете зробити це способом перевірки того, що користувач має роль безпеки **Керівник проекту**.
     - Користувач програми Microsoft Project існує і настроєний належним чином.
 
3. Якщо цей користувач не існує, створіть запис нового користувача. 
4. Виберіть розділ **Нові користувачі**, змініть форму для введення на **Користувач програми**, потім додайте **Ідентифікатор програми**.

   ![Відомості про користувача програми.](media/applicationuserdetails.jpg)


## <a name="issue-4-changes-arent-saved-when-you-create-update-or-delete-a-task"></a>Проблема 4: зміни не зберігаються під час створення, оновлення або видалення завдання

При внесенні одного оновлення до WBS або більше зміни не вносяться й не зберігаються. На сітці розкладу відбувається помилка й відображається повідомлення: «Останню внесену вами зміну зберегти не вдається».

### <a name="mitigation-1-validate-the-license-assignment"></a>Засіб пом'якшення 1: підтвердьте призначення ліцензії

1. Переконайтеся, що користувачу призначено правильну ліцензію, а також, що послугу дозволено у відомостях про плани обслуговування ліцензії.  
2. Переконайтеся, що користувач може відкрити **project.microsoft.com**.
    
### <a name="mitigation-2-validation-configuration-of-the-project-application-user"></a>Засіб пом'якшення 2: підтвердження конфігурації користувача програми Project
1. Переконайтеся в тому, що створено користувача програми Project.
2. Застосуйте до користувача такі ролі безпеки:
  
  - Користувач Dataverse або базовий користувач
  - Система Project Operations
  - Проектна система
  - Система подвійного записування в Project Operations. Ця роль є обов’язковою для сценаріїв розгортання на основі ресурсів і відсутності запасів у Project Operations.
