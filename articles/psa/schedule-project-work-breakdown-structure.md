# <a name="schedule-a-project-with-a-work-breakdown-structure-project-service"></a>Планування проекту з робочою структурою проекту (Project Service)

[!include [banner](../includes/psa-now-project-operations.md)]

[!INCLUDE[cc-applies-to-psa-app-1x-2x](../includes/cc-applies-to-psa-app-1x-2x.md)]

Розкладу проекту визначає, яка робота має бути виконана, які ресурси будуть виконувати роботу і терміни в які треба виконати. Розкладу проекту відображає всі роботи, пов'язані з доставку проекту вчасно. Одним з перших кроків на початку проекту є розкладу проекту. Створіть розклад проекту, щоб створити робочу структура проекту.  
  
 Створіть структуру проекту із робочою структурою проекту, яка допоможе вам:  
  
- Розбивайте роботу на легко керовані завдання  
  
- Підраховуйте час, необхідний для виконання завдання  
  
- Встановлюйте залежності та тривалість завдань  
  
- Визначайте ролі, необхідні для виконання кожного завдання  
  
  Розклад проекту у робочій структурі проекту має знайомий вигляд разом із інтерактивною діаграмою Ганта.  
  
## <a name="create-a-work-breakdown-structure-for-a-project"></a>Створення робочої структури проекту для проекту  
 Створіть робочу структура проекту для представлення послідовності завдань у проекті. Робоча структура проекту включає в себе завданння, вимоги для кожного завдання і дохід, і вартість. У робочу структуру проекту можна додати:  
  
-   Послідовність завдань в ієрархії  
  
-   Інші завдання, якщо такі є, які потрібно завершити, перш ніж завдання може бути запущено  
  
-   Дата початку, дата завершення і тривалість завдання  
  
-   Кількість годин, потрібна для завдання  
  
-   Будь-які необхідні навички та освіта працівників  
  
-   Працівники, які призначені до завдання  
  
-   Орієнтовний дохід та витрати  
  
## <a name="task-types"></a>Типи завдань  
Ви будете використовувати такі типи завдань при створенні робочої структури проекту:  

| | | 
|---------------------------------------|-----------------------------------------------------------------| 
| **Коректний вузол проекту** | Зведене завдання верхнього рівня для проекту. Всі інші завдання проекту створюються під ним. Ім'я кореневого завданням є назва проекту. Зусилля, дати і тривалість кореневого вузла на основі значень в ієрархії під ним. Не можна змінити властивості кореневого вузла або видалити кореневий вузол. | 
| **Підсумок або завдання контейнера** | Зведене завдання – це завдання, яке має субзавдання під ним. Зведене завдання не має жодних власних обсягу роботи для роботи або витрати. Його обсяг роботи та вартість є зведення його субзавдань. Ви можете змінити ім'я зведеного завдання, але не можете змінювати обсяг роботи, дати або тривалість, тому що ті обчислюється автоматично. Видалення зведеного завдання видаляє завдання і всі його субзавдання.|  
| **Завдання кінцевого вузла** | Завдання кінцевого вузла представляє найбільш детальну роботу над проектом. Воно має оцінені обсяг роботи, прогнозовану кількість ресурсів, заплановані дати початку і завершення, і тривалість.|

  
## <a name="task-hierarchy"></a>Ієрархія завдань  
 У вас наступні параметри при створенні ієрархії завдань:  
  
- **Додати завдання**.   Ви можете додати завдання на позицію, яку обираєте в ієрархії завдань. Якщо ви не виберете позицію, нове завдання з'являється в кінці.  
  
- **Знизити рівень завдання**.   Знизьте рівень завдання, щоб зробити його дочірнім завданням безпосередньо над ним.  
  
- **Підвищення рівня завдання**.   Підвищіть рівень завдання, щоб зробити його субзавданням його оригінального батьківського завдання.  
  
- **Вгору та вниз**.   Перемістіть завдання вгору та вниз по ієрархії батьківського завдання. Переміщення завдання вгору або вниз ніяк не впливає на його обсяг роботи, вартість, дати або тривалість.  
  
## <a name="task-attributes"></a>Атрибути завдання  
 Назва завдання описує роботу, яку необхідно закінчити. Використовуйте різні атрибути завдання для опису розкладу і кадрових вимог для завдання.  
  
### <a name="schedule-attributes"></a>Атрибути розкладу

 - Призначте значення до **Години обсягу роботи**, **Кількість ресурсів**, **Дата початку**, **Дата завершення**, і **Тривалість**, щоб визначити розклад для завдання. 
 - **Зусилля** – це оцінка годин, необхідних для завершення завдання.
 - **Кількість ресурсів** – оцінка, яку менеджер проекту ставить у завдання, щоб допомогти вийти з найкращим можливим розкладом. 
 - **Тривалість** (у днях) вказує кількість робочих днів, щоб завершити завдання.  
  
### <a name="staffing-attributes"></a>Атрибути кадрового обслуговування

 - **Роль**, **Організаційну одиницю ресурсу**, **Кількість ресурсів**, і **Ресурси** описують вимоги до комплектування персоналом для завдання. 
 - **Роль** описує тип ресурсу, необхідного для виконання завдання. 
 - **Організаційна одиниця ресурсу** вказує організаційну одиницю, з якої ресурси повинні будуть надані для завдання. Це також впливає на вартість і оцінка збуту завдання, оскільки це вираховується під час визначення ціна одиниці для ресурсу. 
 - **Ресурси** містить загальні ресурси або іменований ресурс, коли знаходиться.  
  
## <a name="task-dependencies"></a>Залежності завдання  
 Ви можете створити зв’язки попередника між одним або кількома завданнями у робочій структурі проекту. Ви можете встановити одне або кілька значень для поля попередника на завданнях для позначення завдань, від яких воно залежатиме. Коли ви призначите значення попередника до завдання, завдання може початися, коли всі завдання-попередники завершилися. Настроївши цю залежність щодо завдання призведе в перерахуванні на запланованої дати початку завдання як останнього кінця всіх його попередників. Впливи пов’язані з попередником у розкладі не обмежені режимом завдання, визначеним щодо завдання.  
  
## <a name="task-mode"></a>Режим завдання  
 Режим завдання є одним з важливих факторів, що визначають планування завдань кінцевого вузла. Є два режими завдань для кожного завдання: режим автопланування і режим ручного планування.  
  
-   **Автоматичне планування**.   Якщо ви встановлюєте режим завдання на Автоматично запланований, двигун планування завдання використовує правила планування для наступних атрибутів завдання, щоб визначити графік завдання:  
  
    -   Попередники  
  
    -   Обсяг роботи  
  
    -   Кількість ресурсів  
  
    -   дата початку та завершення;  
  
-   **Правила планування**.   Дата початку завдання кінцевого вузла, яке не має попередників, за замовчуванням є датою початку планування. Тривалість завдання кінцевого вузла завжди розраховується як кількість робочих днів між датами початку та завершення. Коли автоматично заплановано завдання, ядро планування дотримується наступних правил:  
  
    -   Дати початку та закінчення завдання завжди повинні бути робочими днями відповідно до календаря планування проекту  
  
    -   Дата початку завдання кінцевого вузла, яке має попередників, за замовчуванням є кінцевою датою його попередників  
  
    -   Зусилля = Кількість людей * Тривалість * годин у стандартний робочий день календаря проекту  
  
-   **Планування вручну**.   У деяких випадках ви можете відійти від цих правил. У цих випадках ви можете встановити режим завдання для планування завдання вручну. Це зупиняє ядро планування, щоб воно не розраховувало значення для інших атрибутів планування. Налаштування попередників для завдань завжди впливає на дату початку залежного завдання .  
  
## <a name="create-a-work-breakdown-structure"></a>Створення робочої структури проекту  
  
1.  Перейти до **Project Service > Проекти**.  
  
2.  Натисніть на проект, над яким ви хочете працювати.  
  
3.  В смужці у верхній частині екрану виберіть стрілку вниз біля назви проекту і потім натисніть Структура декомпозиції робіт.  
  
4.  Щоб додати завдання, натисніть кнопку **Додати завдання**. Заповніть поля для завдання та натисніть кнопку **Зберегти**.  
  
5.  Продовжуйте додавання завдань до завершення робочої структури проекту. При створенні робочої структури проекту ви можете виконати таке для впорядкування завдань:  
  
    -   Виберіть завдання та натисніть **Відступ**, щоб перемістити його під інше завдання, або натисніть Outdent, щоб перемістити його на один рівень.  
  
    -   Виберіть завдання та натисніть **Перемістити догори** або **Перемістити донизу**, щоб перемістити його догори або донизу в списку.  
  
    -   Натисніть **Приховати Gantt**, щоб приховати графік Gantt, та натисніть **Показати Gantt**, щоб знову його показати.  
  
    -   Виберіть інший період часу для графіку Gantt у **Шкала часу**.  
  
6.  Для додавання ролей, які ви уточнили у структурі декомпозиції робіт для членів команди вашого проекту, натисніть **Згенерувати команду проекту**.  
  
7.  Коли ви закінчили вносити зміни, натисніть на **Зберегти** в нижньому правому куті екрану.
