# <a name="sample-data-installation-for-the-project-service-application"></a>Інсталяція зразків даних для програми Project Service

[!include [banner](../includes/psa-now-project-operations.md)]

Для спрощення створення власних демонстраційних середовищ, Microsoft надає для завантаження зразки пакетів, які показують можливості ваших програм. Є два типи пактів зразків даних:
- посилання/дані настроювання
- демонстраційні дані (посилання/настроювання й транзакцій дані, такі як наряди-замовлення та проекти)

Зразки пакетів **довідкових** даних можуть завантажуватися в трьох окремих пакетах, тому можна інсталювати дані лише для Project Service або лише для Field Service, або можна інсталювати зразки даних для обох програм одразу.

Пакети зразків даних посилання/дані настроювання:

- [**V902PSMasterData** - тільки версія Project Service 3.x](https://go.microsoft.com/fwlink/?linkid=2026540&clcid=0x409)

- [**V902FSMasterData** - лише версія Field Service 8.x](https://go.microsoft.com/fwlink/?linkid=2026536&clcid=0x409)

- [**V902FPSMasterData** - Field Service 8.x і Project Service 3.x](https://go.microsoft.com/fwlink/?linkid=2026041&clcid=0x409)

Найновіший пакет **демонстраційних** даних включає:

 - [**FPSDemoData** - Field Service 8.x та Project Service 3.x](https://aka.ms/fpsdemodatapackage)

   Інструкції з інсталяції дещо відрізняються у користувачів для створення та настроювання розділу, але решта так само, як і в попередніх [**записах блоґу**](https://aka.ms/fpsdemodatablog). Цей пакет містить скорочений набір демо-даних і займає приблизно 3 години для інсталяції.

Ці пакети зразків даних доступні тільки англійською мовою.

> [!IMPORTANT]
> **Неможливо деінсталювати зразки даних.** Слід використовувати ці пакети лише на демонстраційних, оціночних, навчальних і тестових системах. Зауважте також, що встановлення одного окремого пакету, а за тим встановлення іншого окремого пакету, не підтримується. (Іншими словами, не можна встановлювати **FSMasterData**, а потім – **PSMasterData**, або навпаки.) Якщо у вас виникла потреба у зразках даних для обох застосунків у будь-який момент, слід інсталювати пакет **v902FPSMasterData**.

Після встановлення будь-якого пакету зразків даних, процес інсталяції виконує такі дії:

- Створює або встановлює параметри за замовчуванням для використання Project Service, Field Service або обох програм (якщо застосовується).

- Імпортує зразки даних для програм, такі як доступні для резервування ресурси, ролі програми, продажі і прайси вартості, організаційні підрозділи, записи процесу збуту та інші сутності, щоб продемонструвати основні можливості.  

З пакетом **демо даних** ви отримуєте те, що вище, і додаткові транзакцій дані , такі як наряди-замовлення та проекти.

Цікаво, які можливості можна демонструвати за допомогою зразків даних? Див. нижче вигаданий сценарій Fabrikam Robotics в області [технічні примітки](#technical-notes).

Якщо у вас є запитання щодо встановлення пакетів зразки даних, [надішліть повідомлення електронною поштою на fpsdemodata@microsoft.com](mailto:fpsdemodata@microsoft.com).

## <a name="requirements"></a>Вимоги

Протокол встановлення передає такі відомості про цільову інсталяцію (org):

- Основна мова – англійська, а основна грошова одиниця – долар США (USD,$).

- Організація не містить даних Project Service або Field Service, або містить кістяк даних за замовчуванням для кожної нової організації.

- Правильну версію бізнес-додатку вже встановлено:
       
    - **Для FPSDemoData або v902FPSMasterData:** Організація має інстальовані Field Service версії 8. x і Project Service версії 3. x.

    - **Для v902PSMasterData:** Організація має встановлену Project Service версії 3. x.

    - **Для v902FSMasterData:** Організація встановила Field Service версія 8.x.

> [!NOTE]
> Якщо потрібно встановити зразки даних поверх існуючих даних експериментального або демо-середовища Project Service та Field Service (не рекомендовано), необхідно призупинити попереджувальні заходи безпеки, які виконує встановлювач. Для додаткової інформації див. Технічні примітки нижче.

## <a name="prepare-for-installation"></a>Підготовка до встановлення

Потрібно запустити встановлювач на комп'ютері з останньою версією Windows (бажано Windows 10).

Комп'ютер для встановлення має залишатись підключеним до мережі протягом **1 години** для **даних настроювання/довідкових**. (Зазвичай інсталяція забере приблизно 30 хвилин для **FPSMasterData**, яка включає приклади даних для обох застосунків.) Для на **FPSDemoData**, інсталяція забере приблизно **3 години**.

Функція екранної заставки має бути вимкнена. В іншому разі, облікові дані сесії встановлення можуть бути втрачені, коли ввімкнеться функція заставки (якщо ви не в активному режимі протягом часу встановлення).

## <a name="download-and-unpack"></a>Завантаження і розпакування

Встановлювач зразків даних Project Service та Field Service надається з можливістю саморозпаковування. Імена файлів можуть відрізнятися залежно від пакета зразків даних, але в будь-якому випадку кроки однакові для кожного встановлюваного пакета.

Після завантаження пакета, запустіть .exe-файл, а потім прийміть умови для розпакування, стиснуті у zip-файл. Далі необхідно розпакувати вміст файлу у вибрану папку на комп’ютері.

Залежно від операційної системи та налаштувань безпеки, можливо, після розпакування zip-файлу, необхідно буде виконати такі дії:

1. Знайдіть і клацніть правою кнопкою миші на файлі **FPSDemoData.dll** у папці **v902FPSMasterData** / **PackageDeployer_FPSDemoData**.

2. Виберіть елементи **Розблокувати**.

3. Виберіть **Застосувати**.

4. Виберіть **ОК**.


## <a name="create-or-configure-users"></a>Створіть або налаштуйте користувачів

У пакеті **FPSDemoData** потрібно шість користувачів, тоді як **FPSMasterData** потребує одного користувача. Див. правильний, щоб побачити свій пакет зразків даних.

## <a name="create-or-configure-users---setupreference-data-packages"></a>Створення або настроювання користувачів - пакети даних настроювання/посилання

Пакет **FPSMasterData** встановлюється з одним користувачем з іменем Степан Пузяк з описаними тут налаштуваннями. Для встановлення пакету належним чином, вам потрібно створити (або тимчасово перейменувати) користувачів у вашому середовищі відповідно до вхідних зразків даних конфігурації.

Для створення або налаштування користувачів, виберіть **Налаштування** > **Безпека** > **Користувачі**, і виконайте такі дії:

1. Виберіть UserFullname="Степан Пузяк" з іменем користувача "stepanp" (**нижній регістр**) для ролей Project Manager та Practice Manager.

2. Виберіть користувача **Степан Пузяк**, а потім виберіть **Керувати ролями**. Знайдіть і виберіть роль **Системний адміністратор** або виберіть **OK**, щоб надати всі права адміністрування Степану Пузяку. Цей крок необхідний для того, щоб переконатись, що зразки записів створені з коректними правами доступу користувача, а отже правильно заповнені.

3. З завантаженого пакету необхідно оновити файл зіставлення даних за допомогою адреси електронної пошти в контексті користувача за замовчуванням. Щоб зробити це, відкрийте **PkgFolder**, а потім знайдіть і відкрийте файл **ImportUserMapFile.xml** у програмі Блокнот (або Visual Studio або іншому редакторі XML). Встановіть в полі **DefaultUserToMapTo=** адресу електронної пошти користувача Степана Пузяка.

4. Якщо ви не використали Степана Пузяка з іменем користувача **stepanp**, необхідно оновити додаткові файли. Відкрийте файл **DemoDataPreImportConfig.xml**, а потім знайдіть тег **userstocreateandconfigure**. Оновіть тег **\<login\>** іменем користувача Степан Пузяк. Для додаткової інформації див. [технічні примітки нижче](#technical-notes).

## <a name="create-or-configure-users---demo-data-package"></a>Створення або настроювання користувачів - пакет демо-даних

Пакет демо-даних вимагає шість користувачів. Для правильної інсталяції пакету виконайте такі дії:

 1. Створіть або тимчасово перейменуйте наявних користувачів відповідно до конфігурації вхідних прикладів даних, перейшовши за посиланням **Настройки** > **Безпека** > **Користувачі**.
 
    Ці ролі лише потрібні для персоніфікованих демо-даних.  
    - Повне ім’я користувача = "Дмитро Вовк" як виїзний технік Field Service   
    - Повне ім’я користувача = "Данило Чуб" як диспетчера Customer Service та Field Service   
    - Повне ім’я користувача = "Ганна Коваль" як менеджерка облікового запису   
    - Повне ім’я користувача = "Сергій Малий» як менеджера проектів та практичних заходів  
    - Повне ім’я користувача = "Вероніка Прудка" як член робочої групи   
    - Повне ім’я користувача = "Василь Карабін"
  
2. З метою імпортування демо-даних призначте шість користувачів більше ролі адміністратора, щоб зразки записів імпортувати належним чином. 

3. Відкрийте **PkgFolder**, а потім знайдіть і відкрийте **ImportUserMapFile.xml**. Оновіть поля **Створити =** для адреси електронної пошти відповідних користувачів у системі.

   > [!div class="mx-imgBorder"]
   > ![Знімок екрана UserMapFile.](media/sample-data-7.png)

4. Якщо повне ім'я користувача Сергій Малий має інший ідентифікатор ніж **"serhiym"**, то потрібно оновити додатковий файл. Відкрийте файл **DemoDataPreImportConfig.xml**, а потім знайдіть тег **userstocreateandconfigure**. Оновіть тег **\<login\>** тегом loginId (з урахуванням регістру). 

5. Календар першого користувача (у тезі **userstocreateandconfigure**) використовується для заповнення робочого часу для всіх доступних для резервування ресурсів в імпорті демо-даних. Перейдіть до **Настройки** > **Безпеки** > **Користувачі**, знайдіть користувача Сергій Малий і відкрийте параметр «Робочий час». Змініть наявний робочий час, вибравши **Весь повторюваний розкладу на тиждень від початку до кінця**. Переконайтеся, що **Робочий час задано для 8 ранку - 5 вечора (9 годин), з понеділка до п'ятниці і пояс з тихоокеанським часом (США й Канада)**. Це необхідно для забезпечення, що панелі проекту і розкладу показують час, як очікувалося.

**Рекомендація:** Створіть резервну копію організації для випадку необхідності повернутись до початкових даних, якщо під час встановлення зразків даних щось піде не так. Докладніше див. [Резервне копіювання і відновлення прикладів](/dynamics365/customer-engagement/admin/backup-restore-instances).

## <a name="run-the-package-deployer"></a>Запустіть Package Deployer

1. Знайдіть і запустіть **PackageDeployer.exe** у папці **v902FPSMasterData** АБО **PackageDeployer_FPSDemoData**.

2. Прийміть умови та положення.

3. В наступному вікні:

   a. Виберіть тип розгортання **Office 365**.

   б. Використайте користувача та пароль системного адміністратора, налаштованого в "Створення або налаштування користувачів" ("Степан Пузяк" ім'я користувача "stepanp").

   в. Переконайтеся, що обрано **Відобразити список доступних організацій**.

      > [!div class="mx-imgBorder"]
      > ![Знімок екрана вікна Package Deployer з перевіркою "Відображення списку вибраних організацій"](media/sample-data-2.png)

4. Виберіть організацію, куди потрібно встановити зразки даних.

5. Виберіть **Далі**, доки не відобразиться діалогове вікно **Налаштування демо-даних**.

   > [!div class="mx-imgBorder"]
   > ![Знімок екрана статусу вікна встановлювача демо-даних.](media/sample-data-3.png)

6. Перед виконанням цієї процедури, зверніть увагу, що встановлення зразків даних може тривати до години (зазвичай 10 хвилин). Необхідно переконатися, , що комп'ютер увімкнено та підключено до мережі протягом усього процесу встановлення, і що ваш сеанс залишається активним.   

7. Коли будете готові, натисніть кнопку **Далі**, щоб запустити процес встановлення взірців даних. Після завантаження зразків даних натисніть кнопку **Завершити**.

## <a name="verify-the-sample-data-installation"></a>Перевірте встановлення зразків даних

Для контролю коректності переконайтесь, що кількість записів і типів сутностей, перелічених у вигаданому сценарії Fabrikam Robotics відображаються належним чином.

Після повного завантаження зразків даних, увійдіть як користувач Степан Пузяк та підтвердіть наступне:

- Якщо встановлено програму Project Service, перейдіть до **Project Service** > **Налаштуівання** > **Прайс-листи**. Переконайтеся, що курси рахунків та курси цін у взірцях даних встановлено у відповідності до грошової одиниці для кожної країни/регіону.

- Якщо встановлено програму Project Service, перейдіть до **Universal Resource Scheduling** > **Налаштування** > **Організаційні одиниці**. Переконайтесь, що вартість прайса з відповідною валютою була пов'язана з кожним підрозділом організації (за винятком записів міста). Якщо будь-який відсутній, знайдіть та встановіть правильний прайс вартості.

- Якщо встановлено програму Field Service, перейдіть до **Project Service** > **Налаштуівання** > **Прайси**. Переконайтеся, що встановлено ціни рахунків і вартісні ціни. Послідовно виберіть елементи **Field Service** > **Налаштування** > **Прайси** і перевірте, що вартість рахунків і ставки вартості у пакеті даних встановлено відповідно до грошової одиниці для кожної країни/регіону.

## <a name="technical-notes"></a>Технічні примітки

Нижче наведено докладну технічну інформацію про встановлення цих даних.

### <a name="installing-sample-data-on-top-of-existing-data-not-recommended"></a>Встановлення зразків даних понад наявні дані (не рекомендовано)

Примітка: Якщо потрібно встановити зразки даних понад існуючих даних ознайомлювального або демонстраційного середовища Project Service та Field Service (не рекомендовано), необхідно призупинити попереджувальні заходи безпеки, які виконує встановлювач.

Щоб зробити це, у папці **PkgFolder** знайдіть і відкрийте файл **DemoDataPreImportConfig.xml** з Блокноту (або іншого редактора XML).

Знайдіть наступне значення, а потім змініть параметр з істинного на хибне:

```alias
<TerminateOnPreCheckFailure>true</TerminateOnPreCheckFailure>
```

Ця зміна змушує встановлювача обходити деякі важливі перевірки безпеки, зокрема:

- Підтвердження того, що не більше, ніж один активний запис **Організаційний підрозділ**, а потім перейменування його в **Fabrikam US**.

- Підтвердження того, що не більше, ніж один активний запис **Робочого шаблону**.

- Підтвердження того, що існує не більше, ніж один активний запис **Параметри проекту**, а потім перейменування його в **Параметри**.

### <a name="configuration-components"></a>Налаштування компонентів

У цьому файлі конфігурації попереднього імпорту існує безліч інших компонентів конфігурації. Для технічних користувачів деякі з них включають в себе:

- **\<RequiredSolutions\>** вказує необхідні умови встановлення та їх номери версій.

- **\<InstallSampleData\>** контролює, чи встановлені готові зразки даних для програм.

    - хибне - пропускає встановлення цих вбудованих даних (що є змінними)

    - Істина - встановлює вбудовані дані одночасно з встановленням зразків даних FS та PSA

- **\<PreImportDataCollection\>** вказує плановий файл зіставлення даних та пов'язані записи для імпорту перед встановленням основних зразків даних.

- **\<EntitiesToEnableScheduling\>** указує, які сутності слід увімкнути для резервування в Microsoft Dynamics Scheduling (також відомій, як Universal Resource Scheduling).

- **\<UsersToCreateAndConfigure\>** визначає резервні ресурси, які буде створено (якщо вони ще не існують) перед імпортом зразків даних. Зауважте, що зразок обраної системи джерела планованого ресурсу відповідає запису цільової системи планованого ресурсу з повним іменем та логіном кожного ресурсу. Таким чином, не можна змінювати імена в файлі конфігурації, якщо було імпортовано взірці даних у цільовій системі за допомогою цих імен, а тоді змініть назву планованих ресурсів на потрібну відповідно до доступних користувачу записів, а потім експортуйте дані ще раз для імпорту до кінцевої системи призначення (оновлення **ImportUserMapFile.xml** старих та нових записів відповідно).

- **\<PluginsToDisable\>** визначає окремі дискретні компоненти plug-ins, які повинні бути відключені під час імпорту зразків даних, а потім знову запущені.

### <a name="change-the-work-hours-for-sample-resources"></a>Змініть години роботи для зразкових ресурсів

За замовчуванням усі резервні ресурси мають 24 робочі години.

Якщо потрібно змінити робочі години для зразків резервних ресурсів, перейдіть до **Universal Resource Scheduling** > **Планування** > **Ресурси**.

Виберіть користувача (наприклад, Степан Пузяк) та змініть робочі години Пузяка, які будуть застосовуватися різними користувачами. Послідовно виберіть елементи **Universal Resource Scheduling** > **Налаштування** > **Шаблони робочого часу** та відредагуйте запис **Шаблон робочого часу за замовчуванням**. У полі **Шаблон ресурсів** виберіть користувача з робочими годинами, які потрібно застосувати до інших ресурсів. Виберіть **Universal Resource Scheduling** > **Планування** > **Ресурси** > **Активні плановані ресурси**. Виберіть ресурси, які потрібно змінити, а потім виберіть **Встановити календар**. У діалоговому вікні **Шаблон робочого часу** виберіть шаблон **Робочий час за замовчуванням** з правильним шаблоном ресурсу. Під час переходу до панелі розкладів, ви повинні побачити, що робочий час ресурсів оновлено.
