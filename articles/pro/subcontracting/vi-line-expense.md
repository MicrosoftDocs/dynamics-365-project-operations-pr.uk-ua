# <a name="vendor-invoice-lines-for-expense-categories"></a>Позиції в рахунку постачальника для категорій витрат

_**Застосовується до:** розгортання Lite: від угоди до рахунків-проформ_

Рахунок постачальника у Microsoft Dynamics 365 Project Operations може мати позиції рахунків постачальників для категорій витрат. Керівники проектів можуть використовувати позиції рахунків постачальників для категорій витрат, щоб записувати вартість послуг, що надходять як категорії витрат.

Позиції рахунків постачальників для категорій витрат можуть або не можуть посилатися на субпідрядну сервісну роботу для категорій витрат. Якщо позиція рахунка постачальника для категорій витрат посилається на субпідрядний сервісний договір, керівники проектів зможуть зіставляти та перевіряти категорії витрат, за які виставляється рахунок постачальником у витратах, що записані в цих категоріях витрат і затверджені керівниками проекту в проекті.

Наведена таблиця містить відомості про поля позицій рахунку постачальника для категорій витрат.

| Поле | Опис | Функціональний вплив |
| --- | --- | --- |
| Ім'я | Назва позиції рахунка постачальника, яке допоможе з ідентифікацією. | Це ім’я буде відображатися як перший стовпець у всіх полях підстановки на основі позицій рахунку постачальника. |
| Опис | Стислий опис послуг, на які виставляється рахунок постачальником у позиції рахунку постачальника. | Не надано |
| Субпідрядний сервісний договір | Субпідрядний сервісний договір, в якому спочатку були замовлені послуги. | Якщо для рахунка постачальника вибрано субпідрядний сервісний договір, усі позиції в рахунку постачальника успадкують цей вибір. Рахунок постачальника не може мати позиції, які посилаються на різні субпідрядні сервісні договори. |
| Субпідрядна сервісна робота | Субпідрядна сервісна робота, в якій спочатку були замовлені послуги. Список субпідрядних сервісних робіт, які можна вибрати, обмежується позиціями вибраного субпідрядного сервісного договору. | Якщо в позиції рахунка постачальника для категорій витрат вибрано субпідрядну сервісну робота, для полів **Проект**, **Завдання** і **Категорія транзакцій** вводяться стандартні значення з відповідних полів у субпідрядній сервісній роботі. Якщо вибрана субпідрядна сервісна робота містить значення в полях **Завдання проекту**, **Категорія транзакцій** та **Проект**, значення відповідних полів у позиції рахунку постачальника не можуть відрізнятися від цих значень. |
| Дата транзакції | Дата, коли фактичні дані вартості позиції рахунка постачальника буде записана в проекті. |Не надано |
| Клас транзакції | Виберіть **Витрати**, щоб записати рахунок постачальника для категорії витрат. | Значення **Витрати** вказує на те, що позиція рахунка постачальника використовується для запису суми рахунку на послуги, які надходять як категорії витрат. |
| Project | Назва проекту, в якому було використано послуги, на які виставляється рахунок. | Це поле є обов’язковим і не може бути пустим. |
| Завдання | Назва завдання проекту, в якому було використано послуги, на які виставляється рахунок. Це поле доступне, лише якщо вибрано проект. Вибір завдання проекту є необов’язковим. | Якщо це поле пусте, керівник проекту може зіставити позицію рахунка постачальника із записаними витратами в будь-якому завданні проекту. Якщо в позиції рахунка постачальника не міститься посилання на субпідрядну сервісну роботу, та це поле залишиться пустим, фактичні дані вартості, створені позицією рахунка постачальника, не буде зв’язана з фактичними даними збуту, на які не виставлено рахунок. У цьому разі, якщо налаштовано виставлення рахунків на основі завдань, може бути неможливо виставити рахунок на витрати кінцевому клієнту. |
| Категорія транзакції | Категорія транзакцій, за якою виставляється рахунок. Для вибраної категорії транзакцій потрібно створити відповідну категорію витрат. | Комбінація значень **Категорія транзакцій** і **Одиниця** буде використовуватися як значення за замовчуванням або обчислюватиметься для поля **Ціна за одиницю** в позиції рахунку постачальника. |
| Кількість | Уведіть кількість, за яку виставляється рахунок постачальника в цій позиції рахунка. |Не надано|
| Група одиниць вимірювання | Стандартне значення вводиться на основі групи одиниць вимірювання вибраної категорії транзакцій. | Не надано |
| Одиниця вимірювання | Стандартне значення — це базова одиниця із вибраної групи одиниць вимірювання. Це значення можна змінити, щоб придбати будь-яку одиницю цієї групи одиниць. | Комбінація значень **Категорія транзакцій** і **Одиниця** буде використовуватися як значення за замовчуванням або обчислюватиметься для поля **Ціна за одиницю** в позиції рахунку постачальника. |
| Ціна за одиницю | У ціні за одиницю за замовчуванням використовується комбінація значень **категорія транзакцій** та **Одиниці** з проектного прайса, що застосовуються до дати транзакції у позиції рахунку постачальника. | Якщо ціну для застосовного прайса проекту ціну задано в одиниці вимірювання, яка відрізняється від одиниці в позиції рахунку постачальника, система використовує перетворення одиниць вимірювання для обчислення ціни за одиницю. |
| Проміжний результат | Це поле лише для читання обчислюється як *Кількість* &times; *Ціна за одиницю*, якщо значення вводяться в поле **Кількість** і поле **Ціна за одиницю**. Якщо одне або обидва поля є порожніми, в цьому полі ви можете ввести значення.| Не надано |
| Податок із продажу | Введіть суму податку з продажу. | Не надано |
| Загальна сума | Загальна сума позиції рахунку постачальника включно з податками. Це поле обчислюється як *Проміжна сума* + *Податок з продажу*. | Не надано |
