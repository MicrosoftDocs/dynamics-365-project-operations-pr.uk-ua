# <a name="vendor-invoice-lines-for-expense-categories"></a>Позиції в рахунку постачальника для категорій витрат

_**Застосовується до:** розгортання Lite: від угоди до рахунків-проформ_

Рахунок-фактура постачальника в корпорації Майкрософт Dynamics 365 Project Operations може містити рядки рахунків-фактур постачальників для категорій витрат. Керівники проектів можуть використовувати рядки рахунків-фактур постачальників для категорій витрат, щоб записувати витрати на послуги, які закуповуються, як категорії витрат.

Рядки рахунків-фактур постачальників для категорій витрат можуть посилатися або не посилатися на рядок субпідряду для категорій витрат. Якщо рядок рахунка-фактури постачальника за категоріями витрат посилається на субпідряд, керівники проектів зможуть зіставити та перевірити витрати, які виставляються за лінією рахунка-фактури постачальника щодо витрат, які обліковуються в цих категоріях витрат і затверджуються керівниками проекту на проекті.

У таблиці нижче наведено відомості про поля в рядках рахунків-фактур постачальників для категорій витрат.

| Поле | Опис | Функціональний вплив |
| --- | --- | --- |
| Ім'я | Назва рядка накладної постачальника, щоб допомогти з ідентифікацією. | Це ім’я відображатиметься як перший стовпець у всіх підстановках, які базуються на рядках рахунків-фактур постачальників. |
| Опис | Короткий опис послуг, які виставляються постачальником у рядку рахунків-фактур постачальників. | Не надано |
| Субпідрядний сервісний договір | Субпідряд, за яким спочатку були замовлені послуги. | Коли для рахунку-фактури постачальника вибрано субпідрядник, усі рядки в рахунку-фактурі постачальника успадковують цей вибір. Рахунок-фактура постачальника не може містити рядки рахунків-фактур постачальників, які посилаються на різні субпідряди. |
| Лінія субпідряду | Лінія субпідряду, за якою були замовлені послуги. Перелік ліній субпідряду, які можна обрати, обмежується лініями на обраному субпідряді. | Якщо в рядку рахунка-фактури постачальника для категорій витрат вибрано рядок субпідряду, значення за промовчанням для **полів «** Проект»,**·** «Завдання»**та**«Трансакція» вводяться з відповідних полів у рядку субпідряду. Якщо вибраний рядок субпідряду містить значення в **полях «Проект»,** **«Завдання** проекту» та **«Категорія** транзакції», значення відповідних полів у рядку рахунка-фактури постачальника не можуть відрізнятися від цих значень. |
| Дата транзакції | Дата, коли фактична вартість рядка накладної постачальника буде зафіксована на проекті. |Не надано |
| Клас транзакції | Виберіть **елемент Витрати**, щоб записати рахунок-фактуру постачальника для категорії витрат. | **Значення Витрати** вказує на те, що рядок рахунка-фактури постачальника використовується для запису суми рахунку-фактури за послуги, які були придбані, як категорії витрат. |
| Project | Назва проекту, на якому використовувалися послуги, яким виставляється рахунок-фактура. | Це поле є обов’язковим для заповнення, і його не можна залишати порожнім. |
| Завдання | Ім’я завдання проекту, для якого використовувалися служби, для яких використовується рахунок-фактура. Це поле доступне, лише якщо вибрано проект. Вибір завдання проекту не є обов’язковим. | Якщо це поле залишити порожнім, керівник проекту може зіставити рядок рахунка-фактури постачальника з витратами, які записуються на будь-яке завдання проекту. Якщо рядок рахунка-фактури постачальника не посилається на рядок субпідряду, а це поле залишається пустим, фактична вартість, створена рядком рахунка-фактури постачальника, не буде пов’язана з будь-якими необлікованими фактичними даними про продаж. У цьому випадку, якщо настроєно виставлення рахунків на основі завдань, можливо, витрати не вдасться виставляти кінцевому клієнту. |
| Категорія транзакції | Категорія транзакції, яка виставляється на рахунок-фактуру. Для обраної категорії операцій повинна бути створена відповідна категорія витрат. | Комбінація значень **трансакцій** і **значень одиниць** виміру використовуватиметься як значення за замовчуванням або обчислене значення для **поля Ціна** за одиницю товару в рядку рахунка-фактури постачальника. |
| Кількість | Введіть кількість, яку виставляє постачальник, у рядку рахунка-фактури. |Не надано|
| Група одиниць вимірювання | Значення за замовчуванням вводиться на основі групи одиниць обраної категорії транзакцій. | Не надано |
| Одиниця вимірювання | Значення за замовчуванням – це базова одиниця вибраної групи одиниць. Змінити це значення можна для покупки в будь-якому підрозділі групи одиниць. | Комбінація значень **трансакцій** і **значень одиниць** виміру використовуватиметься як значення за замовчуванням або обчислене значення для **поля Ціна** за одиницю товару в рядку рахунка-фактури постачальника. |
| Ціна за одиницю | У ціні за одиницю товару за замовчуванням використовується комбінація **категорії** Трансакцій і **значень одиниць** з прейскуранту проекту, яка застосовується до дати транзакції рядка рахунка-фактури постачальника. | Якщо ціна для відповідного прейскуранту проекту встановлюється в одиницю, яка відрізняється від одиниці в рядку рахунка-фактури постачальника, система використовує перетворення одиниці для розрахунку ціни за одиницю товару. |
| Проміжний результат | Це поле, доступне лише для читання, обчислюється як *ціна* за одиницю товару &times;*·*, якщо значення введено як у поле «Кількість», **і** в поле «Ціна за **одиницю товару**». Якщо одне або обидва ці поля пусті, можна ввести значення в це поле.| Не надано |
| Податок із продажу | Введіть суму податку з продажу. | Не надано |
| Загальна сума | Загальна сума рядка рахунка-фактури постачальника, включаючи податки. Це поле обчислюється як *податок із продажів за підсумками* + *·*. | Не надано |
