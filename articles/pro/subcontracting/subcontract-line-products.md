# <a name="subcontract-lines-for-products"></a>Субпідрядні сервісні роботи для продуктів

[!include [banner](../../includes/dataverse-preview.md)]

_**Застосовується до:** розгортання Lite: від угоди до рахунків-проформ_

Субпідрядний сервісний договір у Dynamics 365 Project Operations може мати субпідрядну сервісну роботу для продуктів. Ці позиції дають менеджеру проекту змогу купувати в постачальників продукти, які потім можуть використовувати для завдань проекту.

Виконайте описані нижче кроки, щоб створити субпідрядну сервісну роботу для продуктів у Project Operations.

1. На сторінці переходів виберіть **Субпідрядні сервісні договори**, а потім відкрийте субпідрядний сервісний договір, з яким ви хочете працювати. 
2. На вкладці **Субпідрядні сервісні роботи** виберіть **+ Створити**, щоб створити нову субпідрядну сервісну роботу.
3. На сторінці **Швидке створення** в полі **Клас транзакції** виберіть **Матеріал** та введіть або виберіть потрібні відомості поля. 
4. Виберіть **Зберегти**.

У таблиці нижче наведено відомості про поля на сторінці **Субпідрядна сервісна робота** та на сторінці **Швидке створення**, так як вони важливі для придбання продуктів.

| Поле | Опис |
| ----- | ----------- |
| Унікальне ім'я | Ім’я субпідрядної сервісної роботи. |
| Опис | Стислий опис продуктів, які замовлено за субпідрядною сервісною роботою. |
| Тип позиції | У цьому полі використовується значення за замовчуванням **На основі кількості**. |
| Спосіб виставлення рахунків |  Спосіб виставлення рахунків для субпідрядної сервісної роботи. Розклад рахунків на основі проміжних етапів доступний для способів виставлення рахунків з фіксованою ціною. |
| Клас транзакції | У цьому полі використовується значення за замовчуванням **Час**. Щоб створити субпідрядні сервісні роботи для придбання продуктів в полі **Клас транзакції** виберіть значення **Матеріал**. Цей вибір вказує на те, що субпідрядна сервісна робота використовується для запису придбання продуктів, які будуть використовуватися для проектів. |
| Вибрати продукт | Виберіть, чи зберігається продукт, що купується, в каталозі продуктів або є доданим товаром. |
| Продукт | Вибрати активний продукт із каталогу. Це поле доступне, лише якщо для параметра **Вибрати продукт** встановлено значення **Наявний**. |
| Доданий товар | Введіть ім’я доданого товару. Це поле доступне, лише якщо для параметра **Вибрати продукт** встановлено значення **Доданий**.  |
| Запитана дата здачі | Виберіть потрібну дату доставки для продуктів. Ця дата також використовується для вибору прайса проекту з прайсів проекту, вкладених до субпідрядного сервісного договору. Вартість продукту в субпідрядних сервісних роботах за замовчуванням тоді отримується з цього прайса. |
| Дата здачі за договором | Виберіть дату, на яку призначено в угоді здачу продуктів.  |
| Замовлена кількість | Введіть кількість продуктів, яку придбано в постачальника. Якщо менеджер проекту перевищує цю кількість, відображається попередження. |
| Група одиниць вимірювання | Це значення за замовчуванням використовується лише для продуктів каталогу. Якщо вибрано і **Продукт**, і **Запитана дата здачі**, система вибирає застосовний прайс на основі дати здачі. Пов’язані позиції прайса запитуються для відповідного продукту. Значення одиниць вимірювання та групи одиниць вимірювання за замовчуванням отримуються з налаштування в записі позиції прайса. |
| Одиниця вимірювання | Це значення за замовчуванням використовує одиницю вимірювання в записі позиції прайса. За необхідності її можна змінити на іншу одиницю вимірювання. Комбінація продукту та одиниць вимірювання використовується для встановлення ціни за одиницю за замовчуванням у субпідрядній сервісній роботі для існуючих продуктів каталогу. |
| Ціна за одиницю | Ціна за одиницю за замовчуванням отримується за допомогою використання комбінації продукту та одиниці вимірювання з позицій прайсу проекту, які застосовуються для запитаної дати здачі субпідрядної сервісної роботи.  |
| Проміжний результат | Це поле лише для читання обчислюється як «Ціна за одиницю х Кількість», якщо в обидва поля введено значення. Якщо поле **Кількість**, поле **Ціна за одиницю** або обидва поля пусті, ви можете ввести значення вручну.  |
| Податок із продажу | Введіть значення податку з продажу. |
| Загальна сума | У цьому обчислюваному полі відображається загальна сума субпідрядної сервісної роботи після включення податків. Значення в цьому полі обчислюється як проміжна сума + податок. |
