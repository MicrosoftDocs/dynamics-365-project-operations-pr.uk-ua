# <a name="header-details-for-subcontracts"></a>Відомості про заголовок для субпідрядних сервісних договорів

[!include [banner](../../includes/dataverse-preview.md)]

_**Застосовується до:** розгортання Lite: від угоди до рахунків-проформ_

У цьому розділі описано функціональність, передбачену в заголовку субпідрядного сервісного договору в Dynamics 365 Project Operations.

Коли менеджер проектів планує та виконує проекти, він може використовувати підрядників і купувати продукти та послуги в постачальників. Коли менеджеру проектів потрібно купити продукти або послуги, він може створити субпідрядний сервісний договір у Project Operations.

Щоб створити субпідрядний сервісний договір, виконайте описані нижче кроки.

1. В області переходів виберіть **Субпідрядні сервісні договори** та на сторінці **Субпідрядний сервісний договір** виберіть **Створити**.
2. Уведіть необхідні відомості, а потім виберіть **Зберегти**.

    У наведеній далі таблиці наводиться інформація про поля на сторінці **Заголовок субпідрядного сервісного договору**.

    | Поле | Опис |Функціональний вплив |
    |---|------|---| 
    | Унікальне ім'я | Ім’я субпідрядного сервісного договору. | У всіх розкривних списках субпідрядного сервісного договору найменування субпідрядного сервісного договору зазначається в першому стовпці, щоб вам було легше ідентифікувати субпідрядний сервісний договір. | 
    | Опис | Стислий опис послуг і продуктів, які придбано за субпідрядним сервісним договором. | Не надано |
    | Постачальник | Назва компанії, у якої придбано продукти та послуги. Цей запис бізнес-партнера має тип зв’язку **Постачальник** або **Продавець**. | Залежно від вибраного постачальника значення за замовчуванням вводяться автоматично для наведених далі полів:<br/> **• Валюта** </br> **• Прайси** </br> **• Умови оплати**</br> **• Адреса для оплати**</br> **• Адреса для надсилання рахунків**</br> **• Ім’я для виставлення рахунків** </br>**• Керівник організації-постачальника**|
    | Дата субпідрядного сервісного договору | Дата, коли створено сервісний договір субпідряду. | Дата сервісного договору субпідряду використовується для коригування прайсу придбання відносно прайсів, що додаються до пов’язаного постачальника, або відносно проектних параметрів. |
    | Опис статусу | Стан субпідрядного сервісного договору. | Стан визначає перебування сервісного договору субпідряду в бізнес-процесі або можливість його редагування. <br/>Значення, зокрема є такими:<br>• **Чернетка**: сервісний договір субпідряду можна редагувати. Редагуванню підлягають лише сервісні договори субпідряду зі станом **Чернетка**.<br/>• **Підтверджено**: переговори з постачальником завершені, й сервісний договір субпідряду затверджено для доставки. <br/>• **Закрито**: доставку сервісного договору субпідряду завершено.<br/>• **Скасовано**: сервісний договір субпідряду скасовано, доставка не планується.  | 
    | Грошова одиниця | Валюта, в якій придбаваються продукти та послуги. Значення за замовчуванням вводиться автоматично на основі бізнес-партнера постачальника, але його можна змінити. | Валюта сервісного договору субпідряду використовується для вибору прайсу придбання відносно прайсів, що додаються до пов’язаного постачальника, або відносно проектних параметрів. Прайси в іншій валюті не можна пов’язати з сервісним договором субпідряду. Вартість часу, витрат і матеріалів, які ресурси постачальника доставляють за цим сервісним договором субпідряду, відображаються в цій валюті проекту. Після збереження запису сервісного договору субпідряду, валюта сервісного договору субпідряду не підлягає зміні.|
    | Одиниця для договору | Відділення компанії, що укладає угоду про придбання або субпідрядний сервісний договір із постачальником. | Не надано |
    | Умови оплати | Умови оплати за рахунками-фактурами постачальника, що випускаються за цим сервісним договором субпідряду. Значення за замовчуванням вводиться автоматично на основі запису бізнес-партнера постачальника. | Умови оплати за сервісним договором субпідряду копіюються до всіх рахунків-фактур постачальника, пов’язаних із цим сервісним договором субпідряду. Якщо сервісний договір субпідряду має стан **Чернетка**, умови оплати можна оновлювати. | 
    | Адреса оплати | Адреса постачальника, на яку необхідно надсилати оплату. Значення за замовчуванням вводиться автоматично на основі запису бізнес-партнера постачальника. | Адреса для оплати із сервісного договору субпідряду копіюється як адреса для оплати до всіх рахунків-фактур постачальника, що створюються для цього договору субпідряду. Якщо сервісний договір субпідряду має стан **Чернетка**, адресу для оплати можна оновлювати.|
    | Ім’я для виставлення рахунків | Ім’я особи або назва відділення компанії постачальника, яка надсилатиме рахунок. Значення за замовчуванням вводиться автоматично на основі запису бізнес-партнера постачальника. | Значення **Ім’я для надсилання рахунків** за сервісним договором субпідряду копіюється до всіх рахунків-фактур постачальника, пов’язаних із цим сервісним договором субпідряду. Якщо сервісний договір субпідряду має стан **Чернетка**, це поле можна оновлювати.|
    | Адреса для документів | Це адреса, яка використовується в рахунках-фактурах постачальника. Значення за замовчуванням вводиться автоматично на основі запису бізнес-партнера постачальника. | Ця адреса – це адреса «рахунок-фактура від», яка зазначається на рахунках-фактурах постачальника, що створюються за цим договором субпідряду. |
    | Проміжний результат | Якщо сервісний договір субпідряду не має рядків, введіть проміжний підсумок замовлення до сплати податків. Якщо у субпідрядному сервісному договору є роботи, це поле доступне лише для читання. Сума, яка відображається, є проміжним підсумком усіх рядків сервісного договору субпідряду. | Не надано |
    | Загальна сума податків | Якщо сервісний договір субпідряду не має рядків, введіть загальну суму податків за цим сервісним договором субпідряду. Якщо у субпідрядному сервісному договору є роботи, це поле доступне лише для читання. Сума, яка відображається, є сумою податків усіх рядків сервісного договору субпідряду. | Не надано |
    | Загальна сума | У цьому обчислюваному полі відображається загальна сума субпідрядного сервісного договору після включення податків. | Не надано |
    | Дата підтвердження | Дата підтвердження сервісного договору субпідряду. | Не надано |
    | Автор запиту | За замовчуванням у цьому полі задано ім’я користувача, який створює сервісний договір субпідряду. Однак особа, яка створює сервісний договір субпідряду, може змінити це значення із зазначенням особи, від імені якої вона створює відповідний сервісний договір субпідряду. | Не надано |
    | Керівник організації-постачальника | Ім’я основної контактної особи постачальника. Значення за замовчуванням вводиться автоматично на основі запису бізнес-партнера постачальника. У якості керівника бізнес-партнера постачальника за сервісним договором субпідряду ви можете вибрати іншу контактну особу. | Ви можете налаштувати оповіщення електронною поштою, щоб повідомити цій контактній особі, коли в результаті цінових переговорів до сервісного договору субпідряду вносяться зміни. |
