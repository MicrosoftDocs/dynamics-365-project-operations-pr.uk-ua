# <a name="header-details-for-vendor-invoices"></a>Відомості про заголовок для рахунків постачальників

_**Застосовується до:** розгортання Lite: від угоди до рахунків-проформ_

У цій статті описано функціональні можливості, які надаються в заголовку рахунка постачальника в Microsoft Dynamics 365 Project Operations.

Коли менеджер проектів планує та виконує проекти, він може використовувати підрядників і купувати продукти та послуги в постачальників. Під час виконання проекту витрати виставляються на послуги, матеріали та категорії витрат, що закуповуються за субпідрядними сервісними договорами з постачальниками. Постачальники виставляють рахунки на ці витрати для проектів, створюючи рахунки постачальників.

Наведена таблиця містить відомості про поля в заголовках рахунку постачальника в Project Operations.

| Поле | Опис | Функціональний вплив |
| --- | --- | --- |
| Ім'я | Ім’я рахунка постачальника. | У всіх розкривних списках рахунку постачальника найменування рахунку постачальника зазначається в першому стовпці, щоб вам було легше ідентифікувати рахунок постачальника. За замовчуванням, коли створюється рахунок постачальника з субпідрядного сервісного договору, в полі **Ім’я** рахунка постачальника встановлюється значення, яке складається з імені субпідрядного сервісного договору та поточної дати. |
| Опис | Стислий опис послуг і продуктів, на які виставляється рахунок постачальником у позиції рахунку постачальника. | Не надано |
| Постачальник | Назва компанії, яка виставляє рахунок на продукти та послуги. Ця компанія має бути записом бізнес-партнера, який має тип зв’язку **Постачальник** або **Продавець**. | <p>Залежно від вибраного постачальника значення за замовчуванням вводяться автоматично в наведені далі поля:</p><ul><li>Грошова одиниця</li><li>Прейскуранти</li><li>Умови оплати</li><li>Адреса оплати</li></ul> |
| Субпідрядний сервісний договір | Посилання на субпідрядний сервісний договір для рахунка постачальника. | <p>Залежно від вибраного субпідрядного сервісного договору стандартні значення вводяться автоматично в наведені нижче поля:</p><ul><li>Грошова одиниця</li><li>Прейскуранти</li><li>Умови оплати</li><li>Адреса оплати</li></ul><p>Субпідрядний сервісний договір, вибраний у заголовку рахунка постачальника, за замовчуванням вводиться в позиції рахунку постачальника та їх не можна змінити.</p> |
| Дата рахунка | Дата фактичних даних вартості, яку буде створено під час підтвердження рахунка постачальника. | Дата рахунка також використовується для коригування прайсу придбання відносно прайсів, що додаються до пов’язаного постачальника, або відносно проектних параметрів. |
| Опис стану | Стан рахунка постачальника. | <p>Стан визначає перебування рахунка постачальника в бізнес-процесі або можливість його редагування. Нижче наведено деякі доступні значення:</p><ul><li>**Чернетка**: рахунок постачальника можна редагувати.</li><li>**Підтверджено**: рахунок постачальника було перевірено та підтверджено. Рахунки постачальника в цьому стані не можна редагувати та видаляти.</li><li>**Виконується**: рахунка постачальника перевіряється. Рахунки постачальника в цьому стані можна редагувати, але не можна видаляти.</li><li>**Скасовано**: рахунок постачальника скасовано. Рахунки постачальника в цьому стані не можна редагувати та видаляти.</li></ul> |
| Грошова одиниця | Грошова одиниця в рахунку постачальника, за якою виставляється рахунок постачальника на продукти та послуги. | У рахунку постачальника, який посилається на субпідрядний сервісний договір, грошова одиниця субпідрядного сервісного договору вводиться за замовчуванням як грошова одиниця рахунка постачальника. У рахунку постачальника, який не посилається на субпідрядний сервісний договір, стандартне значення вводиться із запису постачальника бізнес-партнера та його можна змінити. Після збереження рахунка постачальника грошову одиницю більше не можна редагувати. |
| Одиниця для договору | Відділення компанії, яке відповідає за отримання послуг та/або продуктів від постачальника. | Не надано |
| Умови оплати | Умови оплати за рахунками постачальника, що випускаються. Значення за замовчуванням вводиться автоматично на основі запису бізнес-партнера постачальника. | Умови оплати за сервісним договором субпідряду копіюються до всіх рахунків постачальника, пов’язаних із цим субпідрядним сервісним договором. Якщо рахунок постачальника має стан **Чернетка**, умови оплати можна оновлювати. |
| Адреса оплати | Адреса постачальника, на яку необхідно надсилати оплату. Значення за замовчуванням вводиться автоматично на основі запису бізнес-партнера постачальника. | Адреса для оплати із сервісного договору субпідряду копіюється як адреса для оплати до всіх рахунків-фактур постачальника, що створюються для цього договору субпідряду. Якщо рахунок постачальника має стан **Чернетка**, адресу для оплати можна оновлювати. |
| Проміжний результат | Якщо рахунок постачальника не має рядків, введіть проміжний підсумок рахунку до сплати податків. Якщо в рахунку постачальника є роботи, це поле доступне лише для читання. У цьому випадку сума, яка відображається, є проміжним підсумком усіх позицій в рахунку постачальника. | Не надано |
| Загальна сума податків | Якщо рахунок постачальника не має рядків, введіть загальну суму податків для субпідрядного сервісного договору. Якщо в рахунку постачальника є роботи, це поле доступне лише для читання. У цьому випадку сума, яка відображається, є підсумком суми податків усіх позицій в рахунку постачальника. | Не надано |
| Загальна сума | У цьому обчислюваному полі відображається загальна сума рахунку постачальника після включення податків. | Не надано |

> [!NOTE]
> Після збереження рахунка постачальника в рахунку не можна змінити такі поля: **Постачальник**, **Субпідрядний сервісний договір**, **Грошова одиниця**, **Одиниця для договору** та **Умови оплати**. Якщо у цих полях рахунку постачальника необхідно внести зміни, слід видалити рахунок постачальника та створити новий.
