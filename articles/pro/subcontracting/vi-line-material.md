# <a name="vendor-invoice-lines-for-products"></a>Позиції в рахунку постачальника для продуктів

_**Застосовується до:** розгортання Lite: від угоди до рахунків-проформ_

Рахунок постачальника в Microsoft Dynamics 365 Project Operations може мати позиції в рахунку постачальника для продуктів (які також називають матеріалами). Керівники проектів можуть використовувати позиції рахунків постачальників для продуктів для запису витрат на продукти, придбані в проектах.

Позиції рахунків постачальників для продуктів можуть або не можуть посилатися на субпідрядну сервісну роботу для матеріалів. Якщо позиція рахунка постачальника для продуктів посилається на субпідрядний сервісний договір, керівники проектів зможуть зіставляти та перевіряти продукти, за які виставляється рахунок постачальником у використанні придбаних продуктів, записані та затверджені керівниками проекту.

Наведена таблиця містить відомості про поля позицій рахунку постачальника для продуктів.

| Поле | Опис | Функціональний вплив |
| --- | --- | --- |
| Ім'я | Назва позиції рахунка постачальника, яке допоможе з ідентифікацією. | Це ім’я буде відображатися як перший стовпець у всіх полях підстановки на основі позицій рахунку постачальника. |
| Опис | Стислий опис продуктів, на які виставляється рахунок постачальником у позиції рахунку постачальника. | Не надано |
| Субпідрядний сервісний договір | Субпідрядний сервісний договір, в якому спочатку були замовлені продукти. | Якщо для рахунка постачальника вибрано субпідрядний сервісний договір, усі позиції в рахунку постачальника успадкують цей вибір. Рахунок постачальника не може мати позиції, які посилаються на різні субпідрядні сервісні договори. |
| Субпідрядна сервісна робота | Субпідрядна сервісна робота, в якій спочатку були замовлені продукти. Список субпідрядних сервісних робіт, які можна вибрати, обмежується позиціями вибраного субпідрядного сервісного договору. | Якщо в позиції рахунка постачальника для продуктів вибрано субпідрядну сервісну робота, для полів **Проект**, **Задача** та **Продукт** вводяться стандартні значення з відповідних полів у субпідрядній сервісній роботі. Якщо вибрана субпідрядна сервісна робота містить значення в полях **Проект**, **Задача** та **Продукт**, значення відповідних полів у позиції рахунку постачальника не можуть відрізнятися від цих значень. |
| Дата транзакції | Дата, коли фактичні дані вартості позиції рахунка постачальника буде записана в проекті. | Не надано|
| Клас транзакції | Під час виставлення рахунка на ці продукти, для цього поля необхідно встановити значення **Матеріал**. | Значення **Матеріал** вказує на те, що позиція рахунка постачальника використовується для запису суми рахунку на придбані матеріали. |
| Project | Назва проекту, в якому було використано продукти, на які виставляється рахунок. | Це поле є обов’язковим і не може бути пустим. |
| Завдання | Назва задачі проекту, в якій було використано продукти, на які виставляється рахунок. Це поле доступне, лише якщо вибрано проект. Вибір завдання проекту є необов’язковим. | Якщо це поле пусте, керівник проекту може зіставити позицію рахунка постачальника з придбаним продуктом, що використовується в будь-якому завданні проекту. Якщо в позиції рахунка постачальника не міститься посилання на субпідрядну сервісну роботу, та це поле залишиться пустим, фактичні дані вартості, створені позицією рахунка постачальника, не буде зв’язана з фактичними даними збуту, на які не виставлено рахунок. У цьому разі, якщо налаштовано виставлення рахунків на основі завдань, буде неможливо виставити рахунок на витрати кінцевому клієнту. |
| Вибір продукту | Укажіть, чи є продукт, на який виставляється рахунок, наявним продуктом з каталогу або доданим товаром. | Якщо вибрано субпідрядну сервісну роботу, вводиться стандартне значення з цієї субпідрядної сервісної роботи. |
| Продукт | Виберіть продукт із каталогу. Якщо продукт є доданим товаром, введіть назву продукту. | Це поле використовується для введення стандартних цін на придбання для наявних продуктів. |
| Кількість | Уведіть кількість матеріалу, за який виставляється рахунок постачальника у цій позиції рахунка. | Не надано |
| Група одиниць вимірювання | Виберіть групу одиниць вимірювання, в яких виражено кількість, за якою виставлено рахунок. | Не надано |
| Одиниця вимірювання | Стандартне значення — це базова одиниця із вибраної групи одиниць вимірювання. Це значення можна змінити, щоб оплатити будь-яку одиницю вибраної групи одиниць. | Комбінація значень **Продукт** і **Одиниця** уде використовуватися як значення за замовчуванням або обчислюватиметься для поля **Ціна за одиницю** в позиції рахунку постачальника. |
| Ціна за одиницю | У ціні за одиницю за замовчуванням використовується комбінація значень **Продукт** та **Одиниці** з проектного прайса, що застосовуються до дати транзакції у позиції рахунку постачальника. | Не надано |
| Проміжний результат | Це поле лише для читання обчислюється як *Кількість* &times; *Ціна за одиницю*, якщо значення вводяться в поле **Кількість** і поле **Ціна за одиницю**. Якщо одне або обидва поля є порожніми, в цьому полі ви можете ввести значення. | Не надано |
| Податок із продажу | Введіть суму податку з продажу. | Не надано |
| Загальна сума | Загальна сума позиції рахунку постачальника включно з податками. Це поле обчислюється як *Проміжна сума* + *Податок з продажу*. | Не надано |
