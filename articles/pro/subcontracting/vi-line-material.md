# <a name="vendor-invoice-lines-for-products"></a>Позиції в рахунку постачальника для продуктів

_**Застосовується до:** розгортання Lite: від угоди до рахунків-проформ_

Рахунок-фактура постачальника в Корпорації Майкрософт Dynamics 365 Project Operations може містити рядки рахунків-фактур постачальників для продуктів (які також називаються матеріалами). Керівники проектів можуть використовувати рядки рахунків-фактур постачальників для продуктів, щоб фіксувати витрати на продукти, які були придбані на проектах.

Рядки рахунків-фактур постачальників для продуктів можуть посилатися або не посилатися на лінію субпідряду для матеріалів. Якщо рядок рахунка-фактури постачальника для продуктів посилається на субпідряд, керівники проектів зможуть зіставити та перевірити продукти, які виставлені лінією рахунка-фактури постачальника, на відповідність використанню придбаних продуктів, які записуються та затверджуються керівниками проекту.

У таблиці нижче наведено відомості про поля в рядках рахунків-фактур постачальників для продуктів.

| Поле | Опис | Функціональний вплив |
| --- | --- | --- |
| Ім'я | Назва рядка накладної постачальника, щоб допомогти з ідентифікацією. | Це ім’я відображатиметься як перший стовпець у всіх підстановках, які базуються на рядках рахунків-фактур постачальників. |
| Опис | Короткий опис товарів, які виставляються постачальником у рядку рахунків-фактур постачальників. | Не надано |
| Субпідрядний сервісний договір | Субпідряд, за яким спочатку замовлялася продукція. | Коли для рахунку-фактури постачальника вибрано субпідрядник, усі рядки в рахунку-фактурі постачальника успадковують цей вибір. Рахунок-фактура постачальника не може містити рядки рахунків-фактур постачальників, які посилаються на різні субпідряди. |
| Лінія субпідряду | Лінія субпідряду, на якій була замовлена продукція. Перелік ліній субпідряду, які можна обрати, обмежується лініями на обраному субпідряді. | Якщо в рядку рахунка-фактури постачальника для продуктів вибрано рядок субпідряду, значення за промовчанням для **полів «Проект»,** **«Завдання» та**«**Продукт»** вводяться з відповідних полів у рядку субпідряду. Якщо вибраний рядок субпідряду **містить значення в полях «Проект»,** **«Завдання**» та **«Продукт»,** значення відповідних полів у рядку рахунка-фактури постачальника не можуть відрізнятися від цих значень. |
| Дата транзакції | Дата, коли фактична вартість рядка накладної постачальника буде зафіксована на проекті. | Не надано|
| Клас транзакції | Коли продукція виставляється в рахунок-фактуру, в цьому полі слід встановити значення **Material**. | Матеріал значення **вказує** на те, що рядок рахунка-фактури постачальника використовується для запису суми рахунку-фактури для матеріалів, які були придбані. |
| Project | Назва проекту, на якому використовувалися продукти, на яких виставляється рахунок- фактура. | Це поле є обов’язковим для заповнення, і його не можна залишати порожнім. |
| Завдання | Ім’я завдання проекту, для якого використовувалися продукти, які виставляються за рахунком-фактурою. Це поле доступне, лише якщо вибрано проект. Вибір завдання проекту не є обов’язковим. | Якщо це поле залишити порожнім, керівник проекту може зіставити рядок рахунка-фактури постачальника з придбаним продуктом, який використовується для будь-якого завдання проекту. Якщо рядок рахунка-фактури постачальника не посилається на рядок субпідряду, а це поле залишається пустим, фактична вартість, створена рядком рахунка-фактури постачальника, не буде пов’язана з будь-якими необлікованими фактичними даними про продаж. У цьому випадку, якщо виставлення рахунків на основі завдань настроєно, витрати не можна буде виставляти кінцевому клієнту. |
| Вибір продукту | Виберіть, чи є товар, який виставляється за рахунком-фактурою, існуючим товаром з каталогу або товаром для запису. | Значення за замовчуванням вводиться з рядка субпідряду, коли вибрано рядок субпідряду. |
| Продукт | Виберіть товар з каталогу. Якщо товар є товаром, що записується, введіть назву товару. | Це поле використовується для введення закупівельних цін за замовчуванням для наявних продуктів. |
| Кількість | Введіть кількість матеріалу, який виставляється постачальником, у цьому рядку рахунка-фактури. | Не надано |
| Група одиниць вимірювання | Виберіть групу одиниць вимірювання одиниці, в якій виражена величина, яка виставляється. | Не надано |
| Одиниця вимірювання | Значення за замовчуванням – це базова одиниця вибраної групи одиниць. Змінити цю величину для оплати можна в будь-якій одиниці обраної групи одиниць. | Комбінація значень **«Продукт»** і **«Одиниця виміру** » використовуватиметься як значення за замовчуванням або обчислене значення для **поля «Ціна** за одиницю товару» в рядку рахунка-фактури постачальника. |
| Ціна за одиницю | У ціні за одиницю товару за замовчуванням використовується комбінація **значень продукту** та **одиниці** товару з прейскуранту проекту, яка застосовується до дати транзакції рядка рахунка-фактури постачальника. | Не надано |
| Проміжний результат | Це поле, доступне лише для читання, обчислюється як *ціна* за одиницю товару &times;*·*, якщо значення введено як у поле «Кількість», **і** в поле «Ціна за **одиницю товару**». Якщо одне або обидва ці поля пусті, можна ввести значення в це поле. | Не надано |
| Податок із продажу | Введіть суму податку з продажу. | Не надано |
| Загальна сума | Загальна сума рядка рахунка-фактури постачальника, включаючи податки. Це поле обчислюється як *податок із продажів за підсумками* + *·*. | Не надано |
