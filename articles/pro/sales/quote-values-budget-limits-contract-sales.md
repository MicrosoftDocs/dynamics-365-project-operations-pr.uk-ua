# <a name="header-details-for-project-quotes"></a>Відомості про заголовок для цінових пропозицій проекту

_**Застосовується до:** розгортання Lite: від угоди до рахунків-проформ_

У цій статті пояснюється інформація, яка застосовується до цінової пропозиції проекту. Вона включає параметри, які впливають на всі позиції цінової пропозиції, а також інформацію про цінову пропозицію, зведену за всіма позиціями, що необхідно для керування ключовими показниками ефективності цінової пропозиції проекту.

У наведеній далі таблиці викладено поля зі зведеною інформацією про цінові пропозиції проектів, які є унікальними для Dynamics 365 Project Operations або мають певні важливі зміни порівняно з ціновими пропозиціями в Dynamics 365 Sales.

| **Поле** | **Розташування** | **Опис** | **Вплив на наступні етапи** |
| --- | --- | --- | --- |
| Ввести | Вкладка «Зведення» (прихована) | Це поле з набором параметрів дозволяє вибрати перелічені нижче варіанти.</br>- На основі робіт (доступно лише якщо інстальовано Project Operations)</br>- На основі товарів (доступно лише якщо інстальовано Project Operations та Sales)</br>- За сервісним обслуговуванням (доступно, якщо встановлено Dynamics 365 Field Service). | У разі використання програми Project Operations для цього поля автоматично встановлюється значення **За типом робіт**. Це класифікує цінову пропозицію як цінову пропозицію на основі проекту. Цінова пропозиція має бути ціновою пропозицією на основі проекту для забезпечення роботи всіх розширень і функцій за проектом. |
| потенційний клієнт | Вкладка "Зведення" | Посилання на компанію клієнта або запис бізнес-партнера. У разі створення цінової пропозиції з потенційної угоди це поле копіюється з відповідного поля потенційної угоди. | Грошова одиниця в ціновій пропозиції проекту за замовчуванням залежить від грошової одиниці клієнта. Однак це можна змінити до збереження цінової пропозиції. |
| Менеджер із роботи з клієнтами | Вкладка "Зведення" | Ім’я менеджера з роботи з клієнтами для цієї угоди. У разі створення цінової пропозиції з потенційної угоди це поле копіюється з відповідного поля потенційної угоди. | Менеджер із роботи з клієнтами відповідає за керування зв’язками з клієнтом до завершення цього проекту. З огляду на запис планованого ресурсу, прив’язаного до менеджера з роботи з клієнтами, значення договірного підрозділу за замовчуванням залежить від цінової пропозиції проекту. |
| Одиниця для договору | Вкладка "Зведення" | Організаційна одиниця, яка відповідає за виконання проекту (або проектів), пов’язаного з цією ціновою пропозицією. У разі створення цінової пропозиції з потенційної угоди це поле копіюється з відповідного поля потенційної угоди. | Договірний підрозділ – це відділення компанії, яке виконуватиме проекти після закриття угоди. Кожний договірний підрозділ має грошову одиницю, яка використовується для звітування про заплановані та фактичні витрати, які виникають під час реалізації проекту. |
| Прайс продуктів | Вкладка "Зведення" | Це прайс, який використовується для встановлення цін за замовчуванням залежно від позицій на основі продуктів у ціновій пропозиції. Список параметрів для цього поля містить список прайсів, у яких грошова одиниця прайса відповідає грошовій одиниці цінової пропозиції. У разі створення цінової пропозиції з потенційної угоди це поле копіюється з відповідного поля потенційної угоди. За замовчуванням значення цього поля потенційної угоди встановлюється із запису бізнес-партнера, однак його можна змінити. | У разі успішності цінової пропозиції значення поля копіюється до проектного договору, який буде створено. |
| Валюта | Вкладка "Зведення" | Вказує на грошову одиницю, яка використовуватиметься для звітування про вартість цієї угоди. Вона також є грошовою одиницею, у якій клієнту виставляються рахунки за успішною угодою. У разі створення цінової пропозиції з потенційної угоди це поле копіюється з відповідного поля потенційної угоди. За замовчуванням значення цього поля потенційної угоди встановлюється із запису бізнес-партнера, однак користувач може його змінити. | Це поле неможливо відредагувати після збереження цінової пропозиції. Використовується для встановлення значень за замовчуванням для прайсів продуктів і проекту в ціновій пропозиції. Грошова одиниця в ціновій пропозиції має відповідати грошовій одиниці в прайсі. |
| Граничне обмеження | Вкладка "Зведення" | Указує на узгоджений максимум кінцевої вартості, на яку погоджується клієнт у рамках цієї угоди. | Цей максимум обчислюється під час виконання та застосовується до всіх позицій і проектів, пов’язаних із цією угодою. |
| Запитана дата доставки | Вкладка "Зведення" | У разі створення цінової пропозиції з потенційної угоди це поле копіюється з відповідного поля потенційної угоди. | Ця дата використовується як кінцева дата для створення розкладів виставлення рахунків. |

| **Поле** | **Розташування** | **Опис** |
| --- | --- | --- |
| Аналіз прибутковості | Вкладка в ціновій пропозиції | На вкладці відображаються наведені нижче показники.</br>- Загальна оплатна вартість.</br></br>- Загальна неоплатна вартість.</br>- Загальний прибуток.</br>- Загальний прибуток (базовий).</br>- Валовий дохід.</br>- Скоригований валовий дохід.|
| Порівняння з очікуванням клієнта | Вкладка в ціновій пропозиції | На цій вкладці відображаються наведені нижче показники.</br>- Прогнозоване завершення.</br>- Запитане завершення.</br>- Бюджет клієнта.</br>- Сума цінової пропозиції. |
| - Аналіз цінової пропозиції. | Вкладка в ціновій пропозиції | На цій вкладці відображаються зведені відомості про основні ключові показники ефективності для цінової пропозиції проекту, які зазначено нижче.</br>- Порівняння очікувань клієнта щодо бюджету та розкладу.</br>- Валовий дохід.</br>- Скоригований валовий дохід. |
