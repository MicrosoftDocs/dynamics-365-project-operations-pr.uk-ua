# <a name="configure-chargeable-components-of-a-project-based-contract-line"></a>Налаштування платних компонентів сервісної роботи за договором на основі проектів

_**Застосовується до:** Легке розгортання - виставлення попередніх рахунків, Project Operations для сценаріїв на основі ресурсів і відсутності запасів_

Сервісна робота за договором на основі проекту містить *включені* та *платні* компоненти.

Включені компоненти — це компоненти, на які поширюється дія перелічених правил:

  - Спосіб виставлення рахунків і розділення клієнтів
  - Граничні обмеження 
  - Параметри частоти виставлення рахунків-фактур, визначені в сервісній роботі за договором на основі проекту

Деякі з включених компонентів можна позначити як платні, використовуючи поле **Тип виставлення рахунків**. Поле **Тип виставлення рахунків** — це набір параметрів, який дає змогу використовувати в контексті сервісної роботи за договором згадані нижче значення.

  - Оплачується
  - Не оплачується

Платні компоненти можуть визначатися для завдань, ролей і категорій транзакцій.

Платність визначається для завдань для сервісних робіт за договором проекту і застосовується до всіх класів транзакцій, включених до роботи. Якщо поле **Включити завдання** для сервісної роботи за договором пусте або має значення **Увесь проект**, вкладка **Платні завдання** буде недоступна.

Платність, визначена для ролей у сервісній роботі за договором проекту, застосовується лише до класу транзакцій **Час**. Якщо поле **Включити Час** для сервісної роботи за договором має значення **Ні**, вкладка **Платні ролі** буде недоступна.

Платність, визначена для категорій транзакцій у сервісній роботі за договором проекту, застосовується лише до класу транзакцій **Витрата**. Якщо поле **Включити Витрати** має значення **Ні**, вкладка **Платні категорії** буде недоступна.

### <a name="update-a-project-task-as-chargeable-or-non-chargeable"></a>Зміна відображення завдання проекту з неоплатного на платне та навпаки

Завдання проекту може бути платним або не платним для певної сервісної роботи за договором, що робить можливим описаний далі сценарій.

Якщо сервісна робота за договором включає **Час** та певне завдання, **T1** пов'язується із нею, як неоплатне. Якщо існує інша сервісна робота за договором, що містить **Витрату**, ви можете пов’язати завдання T1 із сервісною роботою за договором, як неоплатне. Результатом стане те, що весь час, записаний для завдання, буде платним, а усі витрати будуть неоплатними.

На вкладці **Оплачувані завдання** сервісної роботи за договором можна налаштувати тип виставлення рахунків за завданням способом оновлення поля **Тип виставлення рахунків** вкладеної сітки завдань сервісної роботи за договором. Або ви можете оновити поле **Тип виставлення рахунків** у вкладеній сітці налаштування виставлення рахунків за завданням проекту, в якій відображаються сервісні роботи за договором, пов’язані із цим завданням.

### <a name="update-a-role-as-chargeable-or-non-chargeable"></a>Зміна відображення ролі з неоплатної на платну та навпаки

Роль може бути платною або неоплатною для певної сервісної роботи за договором.

Тип виставлення рахунків для ролі можна налаштувати на вкладці **Платні ролі** сервісної роботи за договором. Щоб це виконати, оновіть поле **Тип виставлення рахунків** у вкладеній сітці **Оплачувані ролі**.

### <a name="update-a-transaction-category-as-chargeable-or-non-chargeable"></a>Зміна відображення категорії транзакцій з неоплатної на платну та навпаки

Категорія транзакцій може бути платною або неоплатною для певної сервісної роботи за договором.

Тип виставлення рахунків для транзакції можна налаштувати на вкладці **Платні категорії** сервісної роботи за договором на основі проекту. Щоб це виконати, оновіть поле **Тип виставлення рахунків** у вкладеній сітці **Оплачувані категорії**.

### <a name="resolve-chargeability"></a>Результуюча оплатність

Кошторис або фактичні дані, створені для часу вважаються платними виключно за наведених далі обставин.

   - **Час** включено до сервісної роботи за договором.
   - **Роль** налаштовано як платну в сервісній роботі за договором.
   - **Включені завдання** задані як **Вибрані завдання** в сервісній роботі за договором.
 
 Якщо задовольняються ці три умови, завдання налаштовується як платне. 

Кошторис або фактичні дані, створені для витрат вважаються платними виключно за наведених далі обставин.

   - **Витрати** включено до сервісної роботи за договором
   - **Категорію транзакції** налаштовано як платну в сервісній роботі за договором.
   - **Включені завдання** задані як **Вибране завдання** в сервісній роботі за договором.
  
 Якщо задовольняються ці три умови, **Завдання** налаштовується як платне. 

Кошторис або фактичні дані, створені для матеріалів вважаються платними виключно за наведених далі обставин.

   - **Матеріали** включено до сервісної роботи за договором
   - **Включені завдання** задані як **Вибрані завдання** в сервісній роботі за договором

Якщо задовольняються ці дві умови, **Завдання** налаштовується як платне.
