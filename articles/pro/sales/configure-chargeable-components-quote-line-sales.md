# <a name="configure-the-chargeable-components-of-a-quote-line---lite"></a>Налаштування платних компонентів рядка цінової пропозиції – легка версія

_**Застосовується до:** розгортання Lite: від угоди до рахунків-проформ_

Для позицій цінової пропозиції на основі проекту існує концепція *включених* та *платних* компонентів.

На включені компоненти поширюється дія перелічених нижче правил.

  - Спосіб виставлення рахунків і розділення клієнтів
  - Граничні обмеження 
  - Параметри частоти виставлення рахунків-фактур, визначені в позиції цінової пропозиції на основі проекту

Деякі з включених компонентів можна позначити як платні, використовуючи поле **Тип виставлення рахунків**. Поле **Тип виставлення рахунків** — це набір параметрів, який дає змогу використовувати в контексті позиції цінової пропозиції згадані нижче значення.

  - Оплачується
  - Не оплачується

Платні компоненти можуть визначатися для завдань, ролей і категорій транзакцій.

Платність визначається для завдань для позицій цінових пропозиції і застосовується до всіх класів транзакцій, включених до цієї позиції. Якщо поле **Включити завдання** має значення **Увесь проект** або пусте, вкладка **Платні завдання** недоступна.

Платність визначається для ролей у позиції цінової пропозиції та застосовується лише до класу транзакцій **Час**. Якщо це поле, **Включити Час**, для позиції цінової пропозиції має значення **Ні**, вкладка **Платні ролі** буде недоступна.

Платність, визначена для категорій транзакцій у позиції цінової пропозиції та застосовується лише до класу транзакцій **Витрата**. Якщо це поле, **Включити Витрати**, для позиції цінової пропозиції має значення **Ні**, вкладка **Платні категорії** буде недоступна.

### <a name="update-a-project-task-to-be-chargeable-or-non-chargeable"></a>Зазначення завдання проекту як оплатне чи неоплатне

Завдання проекту може бути платним або неоплатним в контексті певної позиції цінової пропозиції, що робить можливим окреслений далі сценарій.

Якщо позиція цінової пропозиції на основі проекту містить **Час** і завдання **T1**, завдання буде зв'язано з позицією цінової пропозиції як платне. Якщо існує інша позиція цінової пропозиції, що містить **Витрати**, ви можете пов’язати завдання **T1** із сервісною позицією цінової пропозиції, як неоплатне. Результатом стане те, що весь час, записаний для завдання, буде платним, а усі витрати, записані для завдання, будуть неоплатними.

Тип виставлення рахунків за завданням можна налаштувати на вкладці **Оплачувані завдання** рядка цінової пропозиції на основі проекту способом оновлення поля **Тип виставлення рахунків** у вкладеній сітці **Завдання рядка цінової пропозиції**. Або ви можете оновити тип виставлення рахунків для завдання проекту в полі **Тип виставлення рахунків** вкладеної сітки в налаштуваннях виставлення рахунків проекту, де показано рядки цінової пропозиції, пов’язані із завданням.

### <a name="update-a-role-to-be-chargeable-or-non-chargeable"></a>Зазначення ролі як оплатної чи неоплатної

Роль може бути оплачуваною або неоплачуваною у контексті конкретної позиції цінової пропозиції на основі договору.

Тип виставлення рахунків ролі можна налаштувати на вкладці **Оплачувані ролі** рядка цінової пропозиції способом оновлення поля **Тип виставлення рахунків** у вкладеній сітці **Оплачувані ролі**.

### <a name="update-a-transaction-category-to-be-chargeable-or-non-chargeable"></a>Зазначення категорії транзакцій як оплатної чи неоплатної

Категорія транзакцій може бути платною або неоплатною для певної позиції цінової пропозиції.

Тип виставлення рахунків за транзакцією можна налаштувати на вкладці **Оплачувані категорії** рядка цінової пропозиції способом оновлення поля **Тип виставлення рахунків** у вкладеній сітці **Оплачувані категорії**.
