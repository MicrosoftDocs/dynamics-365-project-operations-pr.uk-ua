# <a name="actuals"></a>Фактичні дані 

_**Застосовується до:** Project Operations для сценаріїв на основі ресурсів і відсутності запасів, полегшене розгортання: угоди та виставлення рахунків_

Фактичні дані представляють собою дані про досягнення проекту, які пройшли перегляд і затвердження. Вони створюються в результаті затвердження записів про час, витрати й використання матеріалів, а також проводок у журналі та рахунків.

## <a name="journal-lines-and-time-submission"></a>Рядки у журналі та надсилання часу

Для отримання додаткових відомостей про запис часу див. [Огляд записів часу](../time/time-entry-overview.md).

### <a name="time-and-materials"></a>Час і матеріали

Якщо надісланий запис часу зв’язаний із проектом, який зіставляється з часом та матеріалами сервісної роботи за договором, система створює два рядки журналу, один для вартості, а інший для збуту, на який не виставлено рахунків.

### <a name="fixed-price"></a>Фіксована ціна

Якщо надісланий запис часу зв’язаний із проектом, який зіставляється з фіксованою ціною сервісної роботи за договором, система створює один рядок у журналі для вартості.

### <a name="default-pricing"></a>Стандартна ціна

Логіка для створення цін за замовчуванням міститься в рядку журналу. Значення полів із запису часу копіюються до рядка журналу. Ці значення містять дату транзакції, сервісну роботу за договором, з якою проект зіставляється, і результат валюти з відповідного прайса.

Поля, що впливають на ціноутворення за промовчанням, наприклад **Роль** та **Одиниця ресурсів**, використовуються для визначення належної ціни в рядку в журналі. Можна додавати настроювані поля до запису часу. Якщо ви бажаєте, щоб значення поля заповнялося в рядках фактичних даних, створіть поле в таблицях **Фактичні дані** та **Рядок у журналі**. Використовуйте спеціальний код, для передавання значення вибраного поля з поля «Проводка часу» до поля «Фактичні дані» через рядок у журналі з використанням джерел транзакції. Докладніше про джерела та зв’язки транзакцій див. у розділі [Прив’язування фактичних даних до оригінальних записів](linkingactuals.md#example-how-transaction-origin-works-with-transaction-connection).

## <a name="journal-lines-and-basic-expense-submission"></a>Рядки у журналі та надсилання базових витрат

Для отримання додаткових відомостей про запис витрат див. [Огляд витрат](../expense/expense-overview.md).

### <a name="time-and-materials"></a>Час і матеріали

Якщо надісланий запис базової витрати зв’язаний із проектом, який зіставляється з часом та матеріалами сервісної роботи за договором, система створює два рядки журналу, один для вартості, а інший для збуту, на який не виставлено рахунків.

### <a name="fixed-price"></a>Фіксована ціна

Коли надіслана проводка основних витрат пов’язана з проектом, який зіставляється із сервісною роботою за договором із фіксованою ціною, систем створює один рядок витрат у журналі.

### <a name="default-pricing"></a>Стандартна ціна

Логіка для введення цін за замовчуванням для витрат залежить від категорії витрат. Дата транзакції, сервісна робота за договором, із якою зіставляється проект, і валюта використовуються для визначення відповідного прайса. Поля, що впливають на ціноутворення за промовчанням, наприклад **Категорія транзакції** і **Одиниця** використовуються для визначення відповідної ціни в рядку в журналі. Однак це застосовується, лише якщо методом ціноутворення в прайсі є **Ціна за одиницю**. Якщо методом ціноутворення є **За собівартістю** або **Націнка на собівартість**, ціна, яка вводиться при створенні проводки витрат, використовується для вартості, а ціна в рядку журналу збуту розраховується на основі методу ціноутворення. 

Ви можете додати настроюване поле до проводки витрат. Якщо ви бажаєте, щоб значення поля заповнялося в рядках фактичних даних, створіть поле в таблицях **Фактичні дані** та **Рядок у журналі**. Використовуйте спеціальний код, для передавання значення вибраного поля з поля «Проводка часу» до поля «Фактичні дані» через рядок у журналі з використанням джерел транзакції. Докладніше про джерела та зв’язки транзакцій див. у розділі [Прив’язування фактичних даних до оригінальних записів](linkingactuals.md#example-how-transaction-origin-works-with-transaction-connection).

## <a name="journal-lines-and-material-usage-log-submission"></a>Рядки в журналі й надсилання журналу використання матеріалів

Докладніше про проводку витрат див. у розділі [Журнал використання матеріалів](../material/material-usage-log.md).

### <a name="time-and-materials"></a>Час і матеріали

Коли надіслана проводка в журналі використання матеріалів пов’язана з проектом, який зіставлено з сервісною роботою за договором за часом і матеріалами, система створює два рядка в журналі: один для вартості, другий для збуту, за яким не виставлено рахунки.

### <a name="fixed-price"></a>Фіксована ціна

Коли надісланий журнал використання матеріалів пов’язано з проектом, який зіставляється із сервісною роботою за договором із фіксованою ціною, система створює один рядок витрат у журналі.

### <a name="default-pricing"></a>Стандартна ціна

Логіка введення цін за промовчанням для матеріалу ґрунтується на поєднанні продукту й одиниці. Дата транзакції, сервісна робота за договором, із якою зіставляється проект, і валюта використовуються для визначення відповідного прайса. Поля, що впливають на ціноутворення за промовчанням, наприклад **Код продукту** й **Одиниця**, використовуються для визначення належної ціни в рядку в журналі. Однак це застосовується лише для продуктів, внесених до каталогу. У разі доданого товару ціна, яка вводиться при створенні проводки журналу використання матеріалів, використовується для вартості та ціни збуту в рядках у журналі. 

До проводки **Журналу використання матеріалів** ви можете додати настроюване поле. Якщо ви бажаєте, щоб значення поля заповнялося в рядках фактичних даних, створіть поле в таблицях **Фактичні дані** та **Рядок у журналі**. Використовуйте спеціальний код, для передавання значення вибраного поля з поля «Проводка часу» до поля «Фактичні дані» через рядок у журналі з використанням джерел транзакції. Докладніше про джерела та зв’язки транзакцій див. у розділі [Прив’язування фактичних даних до оригінальних записів](linkingactuals.md#example-how-transaction-origin-works-with-transaction-connection).

## <a name="use-entry-journals-to-record-costs"></a>Використання журналів для запису вартості

Ви можете використовувати журнали записів, щоб записувати витрати або дохід за матеріалами, комісіями, часом, витратами, а також класами транзакції. Журнали можна використовувати для перелічених нижче цілей.

- Виконуйте перенесення фактичних даних із іншої системи до Microsoft Dynamics 365 Project Operations.
- Запис витрат, що сталися у іншій системі. Ця вартість може складатися з витрат на придбання або вартості субпідрядних витрат.

> [!IMPORTANT]
> Програма не перевіряє тип рядка у журналі або відповідні ціни, введені в рядку журналу. Відповідно, використовувати журнали записів для створення фактичних даних повинні лише користувачі, які повністю усвідомлюють та розуміють вплив фактичних даних на проект. Враховуючи вплив, який мають журнали цього типу, слід ретельно вибирати, хто має доступ до створення журналів записів.
