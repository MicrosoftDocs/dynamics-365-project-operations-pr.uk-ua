# <a name="transaction-origins---link-actuals-to-their-source"></a>Походження транзакції - Зв'яжіть фактичні дані з їх джерелом

_**Застосовується до:** Project Operations для сценаріїв на основі ресурсів і відсутності запасів, полегшене розгортання: угоди та виставлення рахунків-проформ_

Записи про початок транзакції створюються для зв'язування фактичних даних із джерелом, таких як записи часу, записи витрат, журнали використання матеріалів і рахунки-фактури проекту.

У наведеному нижче прикладі показано типову обробку записів часу в життєвому циклі проекту Project Operations.

> ![Обробка цілей часу в операціях проекту.](media/basic-guide-17.png)
 
1. Подання запису часу призводить до створення двох рядків журналу: один для витрат і один для безперешкодних продажів.
2. Остаточне затвердження запису часу призводить до створення двох фактичних фактів: один для витрат і один для необмежених продажів.
3. Після створення рахунка-фактури проекту, створюється транзакція для позиції рахунка за допомогою даних, взятих із фактичних показників неоплаченого збуту.
4. У разі підтвердження рахунка-фактури буде створено два нових фактичних фактичні показника: повернення неоплаченого збуту та фактичні дані про неоплачений збут.

Кожна подія в цьому робочому процесі обробки запускає створення записів у сутності походження транзакції, щоб допомогти побудувати трасування зв'язків між цими записами, створеними через запис часу, рядок журналу, фактичні та відомості про рядок рахунка-фактури.
